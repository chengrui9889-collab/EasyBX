# 报销批次 + 台账导出 — 变更任务计划 CR-001

## 1. 变更概述

**标题**：账号默认值独立编辑与管理

#### T-12: 实现 GET/PUT /api/auth/me 后端路由 🔒

**通俗解释**：后端新增两个接口——获取当前用户信息和更新账号默认值。前端登录后可以通过接口拿到默认值，用户修改后也能保存。

**做什么**：
1. 在 `services/auth_service.py` 新增：
   - `get_user_by_id(db, user_id)`：查询 User 记录，返回 `UserResponse`
   - `update_user_defaults(db, user_id, data: UpdateUserDefaultsRequest)`：exclude_unset 更新 User 默认值字段，返回 `UserResponse`
2. 在 `api/auth.py` 新增：
   - `GET /me`：调用 `auth_service.get_user_by_id()`，返回 200 + `UserResponse`
   - `PUT /me`：接收 `UpdateUserDefaultsRequest`，调用 `auth_service.update_user_defaults()`，返回 200 + `UserResponse`
3. 两个端点均通过 `Depends(get_current_user)` + `Depends(get_db)` 鉴权

**涉及文件**：`server/app/api/auth.py`、`server/app/services/auth_service.py`

**参考**：技术方案 §4（GET /api/auth/me、PUT /api/auth/me） → AC-031, AC-032

**依赖**：无（只用现有 User 模型和 Schema）

**验证标准**（TDD RED 阶段）：
- [ ] `GET /api/auth/me` 已登录 → 200，返回 `UserResponse` 含 `default_department` 等 5 个默认值字段
- [ ] `GET /api/auth/me` 未登录 → 401
- [ ] `PUT /api/auth/me` 传入 `{"default_department": "产教融合"}` → 200，返回的 `default_department` 为"产教融合"
- [ ] `PUT /api/auth/me` 再次调用 GET → 返回的 `default_department` 保持更新后的值
- [ ] `PUT /api/auth/me` 只传一个字段 → 其他字段不变
- [ ] `PUT /api/auth/me` 未登录 → 401

#### T-13: 前端用户类型/API/Store 扩展

**通俗解释**：前端知道用户信息里还包含部门、收款人这些默认字段了，可以通过 API 获取和保存。

**做什么**：
1. 扩展 `types/user.ts`：`User` 接口新增 `default_department`、`default_reporter`、`default_payee`、`default_bank_account`、`default_bank_name`（均为 `string | null`）
2. 新建 `api/users.ts`：
   - `getMe()` → `GET /api/auth/me`
   - `updateDefaults(data)` → `PUT /api/auth/me`
3. 扩展 `stores/authStore.ts`：
   - 新增 `user: User | null`  状态
   - 新增 `fetchUser()`：调用 `getMe()` 并保存到 `user`
   - 新增 `updateDefaults(data)`：调用 API 并更新 `user`
   - 登录成功后自动调用 `fetchUser()`
   - 新增 `clearUser()` 在登出时清理

**涉及文件**：`web/src/types/user.ts`、`web/src/api/users.ts`（新建）、`web/src/stores/authStore.ts`

**参考**：技术方案 §4（GET/PUT /api/auth/me） → AC-031, AC-032

**依赖**：T-12

**验证标准**：TypeScript 编译无错误；authStore 的 `user` 字段在登录后不为 null

**涉及文件**：`web/src/pages/UserSettingsPage.tsx`（新建）、`web/src/components/ui/Sidebar.tsx`、`web/src/App.tsx`

#### T-15: 新建批次时加载账号默认值

**通俗解释**：点"新建批次"时，弹窗里部门、报账人、收款人、银行卡号、开户行自动填上账号的默认值。

**做什么**：
1. 修改 `pages/BatchesPage.tsx`：
   - 在 `useEffect` 中（或 `fetchBatches` 时）确保 `authStore.user` 已加载
   - 当用户点击"新建批次"打开 Modal 时，读取 `authStore.user` 的默认值填充到表单初始状态
   - `department` → `user.default_department`
   - `reporter` → `user.default_reporter`
   - `payee` → `user.default_payee`
   - `bank_account` → `user.default_bank_account`
   - `bank_name` → `user.default_bank_name`
   - `report_date` → 当天日期（YYYY-MM-DD 格式）
2. 用户可手工覆盖这些值（与后端 `create_batch` 逻辑一致：显式传入覆盖默认值）

**涉及文件**：`web/src/pages/BatchesPage.tsx`

**参考**：需求文档 §5（AC-001, AC-012, AC-031, AC-032）；技术方案 §5.1

**依赖**：T-13, T-14

**验证标准**：
- [ ] 点"新建批次"→ 弹窗中部门、报账人、收款人、银行卡号、开户行自动填入默认值
- [ ] 报账日期自动填入当天日期
- [ ] 用户修改已填充的字段 → 提交时使用修改后的值（显式传入覆盖默认值）
