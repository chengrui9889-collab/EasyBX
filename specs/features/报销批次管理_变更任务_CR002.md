# 报销批次 + 台账导出 — 变更任务计划 CR-002

## 1. 变更概述

**标题**：添加发票时预览原始发票文件
**变更类型**：微调
**变更原因**：用户在"添加发票"弹窗中选择发票时，无法预览原始发票的图片/PDF文件，难以确认所选发票是否正确
**对应 AC**：AC-033

## 2. 变更范围

### 2.1 现有影响

| 方面 | 影响 |
|------|------|
| 文档 | 需求文档新增 AC-033 |
| 后端 | **无影响**（`GET /api/invoices/{id}/file` 已存在） |
| 前端 | 仅修改 `BatchDetailPage.tsx` — 在可选发票表格行增加预览按钮 + 简单文件预览 Modal |
| 数据库 | **无影响** |
| 已有测试 | **无影响** |

### 2.2 不受影响的部分

- 后端全部逻辑和 205 个测试不变
- 批次 CRUD 全部逻辑不变
- 台账导出逻辑不变
- 账号设置功能不变

#### T-17: 添加发票弹窗支持预览原始发票

**通俗解释**：在"添加发票"弹窗的表格中，每一行右边增加一个小眼睛图标按钮。点击后弹出一个模态框，展示该发票的原始图片或 PDF 文件，方便用户在勾选前确认发票内容。

**做什么**：
1. 在 `BatchDetailPage.tsx` 中：
   - 新增 `previewInvoiceId` 状态（`number | null`）
   - 导入 `Eye` 图标（来自 `lucide-react`）
   - 在可选发票表格每行的"金额"列右侧增加一个"预览"图标按钮（`Eye`）
   - 点击按钮时设置 `previewInvoiceId` 为对应发票的 ID
   - 新增一个独立 `Modal`（小尺寸），标题为"发票预览"
   - Modal 内容使用 `invoicesApi.getFileUrl(previewInvoiceId)` 作为图片/PDF 的 src
   - 图片用 `<img>` 标签渲染，PDF 用 `<iframe>` 渲染（参考 `InvoiceDetailModal` 中第 111-123 行的判断逻辑）
   - 关闭 Modal 时重置 `previewInvoiceId` 为 `null`

2. **注意**：不引入 `InvoiceDetailModal` 组件（它包含编辑/确认/删除等操作按钮，不适用此场景）。直接写一个轻量的文件预览 Modal。

**涉及文件**：`web/src/pages/BatchDetailPage.tsx`

**参考**：需求文档 §5 AC-033

**依赖**：无（只用现有 `invoicesApi` 和 `Modal` 组件）

**验证标准**（TDD 风格 — 手动验收）：
- [ ] 打开"添加发票"弹窗，每行表格右侧可见"预览"按钮（👁️ 图标）
- [ ] 点击预览按钮 → 弹出一个 Modal，标题为"发票预览"
- [ ] Modal 中展示原始发票文件（图片渲染为图片，PDF 渲染为内嵌预览）
- [ ] 点击 Modal 外部或关闭按钮 → Modal 关闭
- [ ] TypeScript 编译无错误（仅 `test/` 目录预存文件除外）
