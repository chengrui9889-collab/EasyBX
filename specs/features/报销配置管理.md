# 报销配置管理 — 功能需求说明书

## 1. 背景与价值

**为什么做**：当前账号设置只支持单一的报销默认值配置，用户在帮不同同事代办报销时，需要反复手动修改报账人、收款人、银行卡等信息，效率低下且容易出错。

**做成什么样**：用户可在账号设置中添加多个报销配置（每个配置包含名称 + 报账部门、报账人、收款人、银行卡号、开户行），新建批次时通过下拉选择快速填充，支持标记默认配置。

**关联**：`specs/features/报销批次管理.md` AC-012（创建批次时默认值填充）的扩展升级

---

## 2. 范围

### 本次做

- 账号设置页面：报销配置列表展示 + 添加/编辑/删除功能
- 每个配置包含：名称（必填）、报账部门、报账人、收款人、银行卡号、开户行（均可选）
- 配置名称唯一性校验
- 默认配置标记功能（新建批次时自动选中）
- 新建批次：下拉选择配置 + 自动填充字段
- 删除配置不影响已创建的批次数据

### 本次不做

- 配置排序/拖拽
- 配置导入导出
- 批量删除配置
- 配置分组/分类

## 4. 需求详述

### 4.1 配置数据结构

每个报销配置包含以下字段：

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| 配置名称 | ✅ 必填 | 用于区分不同配置，不可重复 |
| 报账部门 | 可选 | 默认报账部门 |
| 报账人 | 可选 | 默认报账人姓名 |
| 收款人 | 可选 | 默认收款人姓名 |
| 银行卡号 | 可选 | 默认收款银行卡号 |
| 开户行 | 可选 | 默认开户行名称 |
| 是否默认 | 系统字段 | 是否为默认配置，仅一个可标记 |

### 4.2 账号设置页面交互

**配置列表展示**：
- 每个配置显示为一张卡片/行
- 展示：配置名称 + 报账人 + 收款人（关键字段摘要）
- 默认配置有明显标识（如星标、"默认"标签）
- 每个配置有"编辑"和"删除"操作按钮

**添加配置**：
- 点击"添加配置"按钮，弹出表单
- 填写配置名称（必填）+ 5个可选字段
- 保存后配置出现在列表中

**编辑配置**：
- 点击"编辑"，弹出表单预填充当前值
- 修改后保存更新

**删除配置**：
- 点击"删除"，二次确认后删除
- 若删除的是默认配置，自动将第一个配置设为默认

**标记默认配置**：
- 每个配置卡片上有"设为默认"按钮/开关
- 点击后该配置成为默认，原默认配置取消标记

### 4.3 新建批次交互

**下拉选择器**：
- 顶部增加"报销配置"下拉选择框
- 下拉选项显示配置名称
- 选择后自动填充：报账部门、报账人、收款人、银行卡号、开户行

**默认行为**：
- 若有默认配置 → 自动选中默认配置
- 若无默认配置但有配置 → 自动选中第一个
- 若无任何配置 → 下拉为空，所有字段留空需手动填写

**切换配置**：
- 切换下拉选项时，字段值随配置切换而更新
- 若用户已手动修改过某字段，切换配置时覆盖为新配置的值

### 4.4 数据独立性

- 批次创建时，配置数据复制到批次记录中
- 批次数据与配置数据独立存储
- 删除配置不影响已创建的批次

### 4.5 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 每个配置必须包含名称（必填），其他字段均可选 |
| BR-002 | 配置名称不可重复 |
| BR-003 | 用户可标记一个配置为"默认" |
| BR-004 | 若无标记默认配置，新建批次时自动选中第一个配置 |
| BR-005 | 删除配置不影响已创建的批次数据 |
| BR-006 | 删除默认配置后，自动将第一个配置设为默认 |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001：添加报销配置**
  - **Given** 用户在账号设置页
  - **When** 点击"添加配置"，填写配置名称"程瑞-产教融合"、报账人"程瑞"、收款人"程瑞"、银行卡号"6222..."、开户行"工商银行"，点击保存
  - **Then** 配置出现在配置列表中，显示名称"程瑞-产教融合"和关键字段摘要

- [ ] **AC-002：配置列表展示**
  - **Given** 用户有多个报销配置
  - **When** 进入账号设置页
  - **Then** 配置列表展示每个配置的名称 + 报账人 + 收款人摘要，默认配置有"默认"标识

- [ ] **AC-003：编辑报销配置**
  - **Given** 用户在账号设置页有配置"程瑞-产教融合"
  - **When** 点击编辑，将报账人改为"张三"，点击保存
  - **Then** 配置更新成功，列表中显示新的报账人"张三"

- [ ] **AC-004：删除报销配置**
  - **Given** 用户在账号设置页有配置"张三-研发部"
  - **When** 点击删除，确认删除操作
  - **Then** 配置从列表中消失

- [ ] **AC-005：新建批次自动填充配置**
  - **Given** 用户有配置"程瑞-产教融合"（报账部门=产教融合、报账人=程瑞、收款人=程瑞）
  - **When** 新建批次
  - **Then** 下拉框自动选中该配置，报账部门、报账人、收款人等字段自动填充对应值

- [ ] **AC-006：切换配置更新字段**
  - **Given** 用户有配置A（报账人=程瑞）和配置B（报账人=张三），新建批次时选中配置A
  - **When** 切换下拉选择为配置B
  - **Then** 报账人字段从"程瑞"更新为"张三"，其他字段同步更新

- [ ] **AC-007：标记默认配置**
  - **Given** 用户有配置A和配置B，均未标记默认
  - **When** 点击配置A的"设为默认"按钮
  - **Then** 配置A显示"默认"标识，新建批次时自动选中配置A

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-008：无配置时新建批次**
  - **Given** 用户从未添加任何报销配置
  - **When** 新建批次
  - **Then** 下拉框为空/禁用，报账部门、报账人、收款人等字段全部留空，需手动填写

- [ ] **AC-009：配置名称为空**
  - **Given** 用户在添加配置表单中
  - **When** 未填写配置名称，点击保存
  - **Then** 提示"请输入配置名称"，不允许保存

- [ ] **AC-010：配置名称重复**
  - **Given** 用户已有配置"程瑞-产教融合"
  - **When** 添加新配置时名称也填写"程瑞-产教融合"，点击保存
  - **Then** 提示"配置名称已存在"，不允许保存

- [ ] **AC-011：删除配置不影响已有批次**
  - **Given** 配置A曾用于创建批次X（报账人=程瑞），批次X已保存
  - **When** 删除配置A
  - **Then** 批次X的报账人仍为"程瑞"，数据不受影响，可正常查看和编辑

- [ ] **AC-012：删除唯一配置**
  - **Given** 用户只有一个配置A
  - **When** 删除配置A
  - **Then** 配置列表为空，新建批次时下拉框为空，所有字段留空

- [ ] **AC-013：删除默认配置后自动重设**
  - **Given** 用户有配置A（默认）和配置B，配置A被删除
  - **When** 删除配置A并确认
  - **Then** 配置B自动成为默认配置，显示"默认"标识

- [ ] **AC-014：编辑配置名称为重复名称**
  - **Given** 用户有配置A"程瑞"和配置B"张三"
  - **When** 编辑配置B，将名称改为"程瑞"，点击保存
  - **Then** 提示"配置名称已存在"，不允许保存

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-015：配置名称唯一性** ← BR-002
  - **Given** 用户添加新配置
  - **When** 输入与已有配置相同的名称
  - **Then** 系统阻止保存，提示名称重复

- [ ] **AC-016：仅一个默认配置** ← BR-003
  - **Given** 用户有配置A（默认）和配置B
  - **When** 将配置B设为默认
  - **Then** 配置A取消默认标记，配置B成为默认

- [ ] **AC-017：无默认配置时选第一个** ← BR-004
  - **Given** 用户有配置A、配置B，均未标记默认
  - **When** 新建批次
  - **Then** 下拉框自动选中配置A（列表第一个）

- [ ] **AC-018：批次数据独立存储** ← BR-005
  - **Given** 用配置A创建批次X后，编辑配置A将报账人改为"李四"
  - **When** 查看批次X
  - **Then** 批次X的报账人仍为原值（创建时的值），不受配置修改影响
