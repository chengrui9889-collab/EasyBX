# 概览页 (Dashboard) — 功能需求说明书

## 1. 背景与价值

**为什么做**：概览页（Dashboard/工作台）是用户登录后看到的第一个页面，是 MVP 的收尾环节。它承担着"入口和概览"的职责——让用户一进来就能掌握全局状态，快速找到下一步操作入口。

**做成什么样**：用户登录后看到三个统计卡片（待处理发票数、本月报销总额、进行中批次数）、最近 5 条报销批次和快捷操作入口。新用户看到友好的空状态引导，引导其开始第一次使用。

**关联**：`specs/开发路线图.md` Phase 1.8

---

## 2. 范围

### 本次做

- 新增后端专用统计接口 `GET /api/dashboard/stats`，一次性返回概览页所需数据
- 统计卡片区域：展示待处理发票数、本月报销总额、进行中批次数
- 最近批次区域：展示最近 5 条报销批次（名称/金额/日期），可点击跳转详情
- 快捷操作区域：上传发票、创建批次、导出台账的快速入口按钮
- 新用户空状态引导：当发票数或批次数为 0 时，展示友好的引导卡片，点击可跳转对应页面
- 加载失败降级：统计卡片显示 `——`，不阻断页面渲染

### 本次不做

- 图表/趋势图
- 活动时间线/操作日志
- 通知/提醒
- 用户自定义卡片布局
- 数据导出/打印概览页

## 4. 验收标准 (Acceptance Criteria)

### 4.1 Happy Path（正常流程）

**AC-001**: Given 用户已登录且有待处理发票（status=pending），When 用户访问概览页，Then 统计卡片"待处理发票数"显示正确的数字（如 12），点击可跳转到发票管理页并自动筛选待处理状态。

**AC-002**: Given 用户已登录且当月有已确认的发票记录，When 用户访问概览页，Then 统计卡片"本月报销总额"显示当月所有已确认发票金额的合计（精确到两位小数），格式如 ¥3,280.50。

**AC-003**: Given 用户已登录且有进行中的报销批次（status != archived），When 用户访问概览页，Then 统计卡片"进行中批次数"显示正确数字。

**AC-004**: Given 用户已有报销批次，When 用户访问概览页，Then "最近批次"区域展示最近 5 条批次（按 created_at 倒序），每条显示批次名称、金额合计、创建日期，点击可跳转到对应批次详情页。

**AC-005**: Given 用户已登录，When 用户访问概览页，Then 页面底部/右侧展示三个快捷操作按钮："上传发票"（跳转 /invoices）、"创建批次"（跳转 /batches）、"导出台账"（跳转 /batches）；每个按钮有图标和文字说明。

### 4.2 Edge & Error Cases（边界和异常）

**AC-006**: Given 新用户首次登录且发票数和批次数均为 0，When 用户访问概览页，Then 统计卡片正常显示"0"，同时展示空状态引导卡片，包含友好文案（如"欢迎使用 EasyBX，上传你的第一张发票开始报销之旅"）和"上传发票"按钮，点击跳转发票上传页。

**AC-007**: Given 后端统计接口请求失败（网络异常/服务端错误），When 用户访问概览页，Then 统计卡片显示 `——` 替代数字，不阻断页面渲染，最近批次区域和快捷操作正常展示（使用已有前端数据）。

### 4.3 Business Rules（业务规则）

**AC-008**: Given 任何情况下，When 计算"本月报销总额"，Then 只统计当前自然月内、status 为 confirmed 的发票金额之和，不包括已软删除的发票。

**AC-009**: Given 任何情况下，When 查询"最近批次"，Then 按 created_at 倒序取 5 条，不包含已删除的批次（deleted_at is not null）。

## 5. 数据依赖

| 数据项 | 数据来源 | 获取方式 |
|--------|---------|---------|
| 待处理发票数 | invoices 表，status='pending' | 后端统计接口 |
| 本月报销总额 | invoices 表，status='confirmed'，本月 | 后端统计接口 |
| 进行中批次数 | batches 表，deleted_at is null | 后端统计接口 |
| 最近 5 条批次 | batches 表，按 created_at 倒序 | 后端统计接口 |
| 用户信息 | authStore / 本地 token | 前端现有状态 |
