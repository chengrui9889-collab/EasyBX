# 登录与注册 — 功能需求说明书

## 1. 背景与价值

**为什么做**：当前登录/注册页面仅有最基本的表单骨架，缺少前端校验、Enter 提交、已登录自动跳转、注册后体验衔接、差异化错误提示等体验细节。登录是用户接触产品的第一触点，体验粗糙会拉低整体产品感受。

**做成什么样**：用户打开登录/注册页后，能感受到一个干净、专业、流畅的入口体验——表单自动校验、Enter 即提交、注册成功后无缝衔接登录、错误提示精准可读。

**关联**：`specs/开发路线图.md` Phase 1.2

---

## 2. 范围

### 本次做

- 登录/注册页的双栏视觉设计（左侧品牌区 + 右侧表单区），风格延续 EasyBX 设计语言
- 客户端表单校验（前端校验字段合法性，无效表单不提交）
- Enter 键触发提交
- 已登录自动跳转工作台（跳过登录页）
- 注册成功后跳转登录页，显示绿色成功提示，用户名自动填入
- 区分不同错误类型（网络失败 / 账号错误 / 用户名已存在 / 服务异常），展示不同提示文案
- 提交中按钮禁用 + 文案切换（"登录中..." / "注册中..."）

### 本次不做

- "记住我"功能 — 个人桌面端无需
- 密码重置功能 — 下期考虑
- 验证码 — 个人使用场景暂不需要
- 第三方登录（微信等） — 产品定位不需要
- 登录页入场动画/过渡特效 — 非核心体验

## 4. 需求详述

### 4.1 核心流程

1. 用户访问任意页面 → 若无 token 自动跳转登录页 → 若已有 token 自动跳到工作台
2. 用户在登录页输入用户名和密码 → 前端校验通过后提交 → 后端验证 → 成功存 token 跳工作台
3. 用户点击"注册"链接进入注册页 → 填写用户名/显示名/密码 → 前端校验通过后提交 → 成功跳登录页并显示绿色提示 + 用户名回填
4. 用户输入错误信息 → 后端返回具体错误 → 前端展示对应错误提示文案

### 4.2 交互规则

- 表单提交后按钮立即变为禁用状态，文案切换为"登录中..." / "注册中..."，防止重复提交
- 输入框失焦时不做校验（体验干扰小），校验在点击提交时统一执行
- 按 Enter 键等同于点击提交按钮
- 登录/注册错误提示消失时机：用户开始修改任一字段时清除
- 注册成功提示在用户切换路由或离开登录页时消失

### 4.3 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 用户名长度 2-50 字符，不可为空 |
| BR-002 | 密码长度 6-128 字符，不可为空 |
| BR-003 | 显示名可选，不为空时最长 50 字符 |
| BR-004 | 登录成功后 token 存入 localStorage，键名 `easybx_token` |
| BR-005 | 退出登录时清除 `easybx_token` 并跳转登录页 |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 未登录用户自动跳转登录页
  - **Given** 用户未登录（localStorage 中无 `easybx_token`）
  - **When** 访问任意页面（如 `/dashboard`、`/invoices`）
  - **Then** 自动跳转到 `/login` 登录页

- [ ] **AC-002**: 已登录用户自动跳转工作台
  - **Given** 用户已登录（localStorage 中有有效的 `easybx_token`）
  - **When** 访问 `/login` 或 `/register`
  - **Then** 自动跳转到 `/dashboard` 工作台，不显示登录/注册页

- [ ] **AC-003**: 正确登录跳转工作台
  - **Given** 用户在登录页输入正确的用户名和密码
  - **When** 点击"登录"按钮或按 Enter 键
  - **Then** 按钮变为"登录中..."且不可再点击，后端验证通过后跳转到 `/dashboard`

- [ ] **AC-004**: 注册成功后跳登录页且用户名回填
  - **Given** 用户在注册页输入合法的用户名、显示名、密码
  - **When** 点击"注册"按钮
  - **Then** 按钮变为"注册中..."且不可再点击，成功后跳转到 `/login`，页面顶部显示绿色提示"注册成功，请登录"，登录表单中的用户名字段自动填入刚注册的用户名

- [ ] **AC-005**: Enter 键触发提交
  - **Given** 用户在登录页或注册页，焦点在任一输入框中
  - **When** 按下 Enter 键
  - **Then** 触发表单提交，行为与点击登录/注册按钮完全一致

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-006**: 空字段前端拦截 — 登录页
  - **Given** 用户在登录页，用户名或密码为空
  - **When** 点击"登录"或按 Enter
  - **Then** 表单不提交，空字段下方显示红色提示"请输入用户名" / "请输入密码"

- [ ] **AC-007**: 密码过短前端拦截 — 注册页
  - **Given** 用户在注册页，密码少于 6 个字符
  - **When** 点击"注册"或按 Enter
  - **Then** 表单不提交，密码框下方显示红色提示"密码至少需要 6 个字符"

- [ ] **AC-008**: 登录凭据错误
  - **Given** 用户输入了错误的用户名或密码
  - **When** 提交登录
  - **Then** 显示红色提示"用户名或密码错误"，密码框内容清空，焦点回到密码框，允许重新输入

- [ ] **AC-009**: 注册用户名重复
  - **Given** 用户输入的用户名已被占用
  - **When** 提交注册
  - **Then** 显示红色提示"用户名已存在"，表单其他字段不清空，用户可修改用户名后重新提交

- [ ] **AC-010**: 网络异常
  - **Given** 用户提交登录/注册时发生网络超时或断网
  - **When** 请求失败（无响应或 Network Error）
  - **Then** 显示红色提示"网络连接失败，请稍后重试"，按钮恢复可点击状态

- [ ] **AC-011**: 服务端未知错误
  - **Given** 用户提交后后端返回非预期的错误（非用户名重复、非凭据错误）
  - **When** 请求返回 500 或其他未知错误状态码
  - **Then** 显示红色提示"服务异常，请稍后重试"，按钮恢复可点击状态

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-012**: Token 存储 ← BR-004
  - **Given** 用户登录成功
  - **When** 后端返回 token
  - **Then** token 存入 localStorage，键名为 `easybx_token`，后续所有 API 请求自动携带此 token

- [ ] **AC-013**: 退出登录清除 Token ← BR-005
  - **Given** 用户在任意需认证的页面
  - **When** 点击顶栏"退出登录"按钮
  - **Then** `easybx_token` 从 localStorage 中清除，页面跳转到 `/login`
