# 发票管理前端 — 功能需求说明书

## 1. 背景与价值

**为什么做**：发票管理前端是 EasyBX 的核心交互入口。后端（Phase 1.3）已提供完整的发票上传、OCR 识别、状态流转、回收站等 API（68 个测试全部通过），前端需要将这些能力呈现为直观、流畅的用户界面，让用户能快速完成"上传 → 查看 → 校验 → 确认入库"的发票入库全流程。

**做成什么样**：用户打开发票管理页面，顶部是拖拽/点击上传区，下面是可切换视图（表格/卡片）的发票列表。通过 Tab 栏快速切换状态视图，点击发票进入 Modal 弹窗查看原文件、编辑字段、确认入库。整个过程流畅、状态一目了然。

**关联**：
- `specs/开发路线图.md` Phase 1.4
- `specs/features/发票管理后端.md`（上游需求）
- `specs/features/发票管理后端_技术方案.md`（对接 API）
- `specs/features/发票管理后端_任务规划.md`（后端实现追溯）

## 2. 范围

### 本次做

- 发票上传区（拖拽/点击上传，展示已选文件预览列表，用户确认后批量上传，上传完成后统一反馈结果）
- 发票列表（支持表格视图和卡片网格视图自由切换）
- 状态 Tab 栏（全部 / 识别中 / 待确认 / 已入库）
- 日期范围筛选（可选，筛选后列表自动刷新）
- 手动刷新按钮
- 分页器（每页 20/50/100/200，上一页/页码/下一页，总条数）
- 查看详情弹窗 Modal（左侧文件预览 + 右侧编辑表单）
- 根据发票类型动态展示字段（通用字段 + 按类型特色字段）
- 各状态下的操作行为差异（编辑、保存、确认入库、删除）
- 回收站 Modal（查看已删除发票、恢复、显示剩余天数）

### 本次不做

- 归档状态视图（Phase 3.2）
- 替票关联（Phase 2.2）
- PDF 导出（Phase 2.4）
- 批次勾选关联（Phase 1.7 复用发票列表组件）
- 发票查重

### 4.3 上传达流程

```
用户拖拽/选择文件
  │
  ├─ 展示已选文件预览列表（文件名 + 大小 + 格式校验标注）
  │   * 格式不符 / 超 50MB → 红色标注原因
  │   * 超 20 张 → 只展示前 20 张 + 提示
  │
  └─ 用户确认 → 点击"上传"
      │
      ├─ 按钮置为加载状态
      ├─ 调用 POST /api/invoices/
      ├─ 完成后展示反馈 Toast：
      │   ✅ 成功：N 张上传成功
      │   ⚠️ 跳过：N 张（格式不符/超大小/超出数量）
      │
      └─ 自动切换到"识别中" Tab 查看新发票
```

### 4.4 Tab 与状态映射

| Tab 标签 | 对应后端 `state` | 说明 |
|---------|----------------|------|
| 全部 | 所有（不含 deleted） | 用户总览 |
| 识别中 | `processing` | 刚上传，OCR 进行中 |
| 待确认 | `pending` + `failed` | 需要用户处理 |
| 已入库 | `confirmed` | 已确认入库 |

### 4.5 卡片/行展示内容

| 元素 | 说明 |
|------|------|
| 发票类型图标 | 增值税/高铁/滴滴/飞机/默认，五类图标 |
| 金额 | 最显眼位置，带 ¥ 前缀 |
| 发票日期 | 格式：YYYY-MM-DD |
| 销售方名称 | 单行展示，过长省略 |
| 状态标签 | processing → 蓝色"识别中"（带动画） |
|  | pending → 黄色"待确认" |
|  | failed → 红色"识别失败" |
|  | confirmed → 绿色"已入库" |
| 操作按钮 | "查看/编辑"主按钮 + 删除图标 |

### 4.7 各状态 Modal 行为

| 发票状态 | 表单可编辑 | 保存按钮 | 确认入库按钮 | 删除按钮 |
|---------|-----------|---------|------------|---------|
| `processing` | ❌ 置灰 + 提示"发票正在识别中" | ❌ 隐藏 | ❌ 隐藏 | ✅ 可用（物理删除） |
| `pending` | ✅ 全部字段可编辑 | ✅ 可用 | ✅ 可用（日期+金额校验通过） | ✅ 可用（物理删除） |
| `failed` | ✅ 全部字段可编辑 | ✅ 可用 | ✅ 可用（日期+金额校验通过） | ✅ 可用（物理删除） |
| `confirmed` | ✅ 全部字段可编辑 | ✅ 可用 | ❌ 隐藏或显示"已入库" | ✅ 可用（软删除→回收站） |

### 4.8 回收站设计

- **入口**：列表页面右上角垃圾桶图标，带数量角标（软删除发票数）
- **布局**：独立 Modal 弹窗
- **列表**：表格展示（发票号、金额、日期、删除时间、剩余天数）
- **操作**：每条发票有"恢复"按钮
- **恢复**：点击恢复 → 确认弹窗 → 恢复成功 → 发票回到"已入库" Tab
- **空回收站**：显示"回收站为空"

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 上传区支持拖拽文件
  - **Given** 用户在发票管理页面
  - **When** 从桌面拖拽一张 jpg/png/pdf 格式文件到上传区
  - **Then** 文件出现在已选文件预览列表中，显示文件名和大小，上传按钮为可用状态

- [ ] **AC-002**: 上传区支持点击选择文件
  - **Given** 用户在发票管理页面
  - **When** 点击上传区，在文件选择器中选择多张 jpg/png/pdf 文件
  - **Then** 所有选中文件出现在已选文件预览列表中，每张显示文件名和大小

- [ ] **AC-003**: 上传完成后统一反馈结果
  - **Given** 用户已在预览列表中确认待上传文件
  - **When** 点击"上传"按钮，上传完成后
  - **Then** 页面顶部显示 Toast 通知"X 张上传成功，Y 张跳过"，列表自动切换到"识别中" Tab

- [ ] **AC-004**: Tab 切换过滤列表
  - **Given** 发票管理页面有多个状态的发票
  - **When** 用户点击"待确认" Tab
  - **Then** 列表只展示 state=pending 和 state=failed 的发票，排序按发票日期倒序

- [ ] **AC-005**: 状态标签正确展示
  - **Given** 列表中存在不同状态的发票
  - **When** 用户浏览列表
  - **Then** processing 显示蓝色"识别中"标签（含旋转动画），pending 显示黄色"待确认"，failed 显示红色"识别失败"，confirmed 显示绿色"已入库"

- [ ] **AC-006**: 表格/卡片视图自由切换
  - **Given** 发票列表有数据
  - **When** 用户点击视图切换按钮
  - **Then** 列表在表格视图和卡片网格视图之间切换，当前视图样式高亮显示

- [ ] **AC-007**: 日期范围筛选
  - **Given** 用户在发票管理页面
  - **When** 选择开始日期和结束日期
  - **Then** 列表只展示该日期范围内的发票，筛选条件显眼可见

- [ ] **AC-008**: 手动刷新
  - **Given** 用户在发票管理页面，有 processing 状态的发票
  - **When** 点击刷新按钮
  - **Then** 列表重新从 API 加载最新数据，processing 发票的状态可能已更新为 pending 或 failed

- [ ] **AC-009**: 查看发票详情弹窗 — 文件预览 + 字段展示
  - **Given** 列表中存在一张 pending 状态的发票
  - **When** 用户点击该发票的"查看/编辑"按钮
  - **Then** 打开 Modal 弹窗，左侧展示发票原文件（图片直接渲染/PDF 用 iframe），右侧展示所有 OCR 识别字段（已有值的回填，空字段留空）

- [ ] **AC-010**: 编辑字段并保存
  - **Given** Modal 中发票为 pending 状态，用户修改了发票号码和金额
  - **When** 用户点击"保存"按钮
  - **Then** 字段更新保存成功，发票状态保持 pending，Modal 关闭，列表数据更新

- [ ] **AC-011**: 确认入库 — 校验通过
  - **Given** Modal 中发票为 pending 状态，发票日期已填写、金额 > 0
  - **When** 用户点击"确认入库"
  - **Then** 发票状态更新为 confirmed，Modal 关闭，"确认入库"按钮变为"已入库"外观，列表数据更新

- [ ] **AC-012**: 表格视图分页
  - **Given** 发票数量超过当前每页条数
  - **When** 用户翻页或更改每页条数
  - **Then** 正确显示当前页数据，分页器显示页码和总条数

- [ ] **AC-013**: 卡片视图中查看 confirmed 发票详情
  - **Given** 列表中有 confirmed 状态发票
  - **When** 用户点击"查看/编辑"按钮
  - **Then** Modal 打开，右侧表单全部字段可编辑，"确认入库"按钮不显示或显示为"已入库"置灰状态

- [ ] **AC-014**: processing 状态下打开详情 Modal
  - **Given** 发票状态为 processing
  - **When** 用户点击"查看"按钮
  - **Then** Modal 打开，左侧可预览原文件，右侧表单全部置灰且不可编辑，顶部显示"发票正在识别中，请稍候..."提示

- [ ] **AC-015**: 回收站恢复发票
  - **Given** 回收站中存在一张未过期的软删除发票
  - **When** 用户点击该发票的"恢复"按钮并确认
  - **Then** 发票从回收站消失，重新出现在"已入库" Tab 中，状态恢复为 confirmed

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-016**: 删除 pending 发票 — 确认后物理删除
  - **Given** 列表中有一张 pending 状态的发票
  - **When** 用户点击删除按钮并确认"确定删除这张发票吗？删除后不可恢复"
  - **Then** 该发票从列表中消失，不进入回收站

- [ ] **AC-017**: 删除 confirmed 发票 — 确认后软删除
  - **Given** 列表中有一张 confirmed 状态的发票
  - **When** 用户点击删除按钮并确认"发票将移至回收站，30天内可恢复"
  - **Then** 该发票从列表中消失，回收站图标角标数字 +1

- [ ] **AC-018**: 空列表展示空状态
  - **Given** 当前 Tab 下没有任何发票（如从未上传过发票）
  - **When** 用户查看该 Tab
  - **Then** 展示空状态图标 + "暂无发票，拖拽文件到上方区域开始上传"文案

- [ ] **AC-019**: 上传超过 20 张文件 — 预览列表限制
  - **Given** 用户选择 25 张发票文件
  - **When** 文件出现在上传预览列表中
  - **Then** 只展示前 20 张，底部提示"已超出数量限制，仅上传前 20 张"

- [ ] **AC-020**: 上传区选择格式不符的文件 — 预览列表标注
  - **Given** 用户选择包含 .doc 文件的多个文件
  - **When** 文件出现在上传预览列表中
  - **Then** 格式不符的文件旁红色标注"不支持的文件格式"，上传按钮仍然可用，但上传时该文件会被单独跳过

- [ ] **AC-021**: 回收站 — 恢复已过期的发票
  - **Given** 回收站中某张发票剩余天数为 0（已超过 30 天恢复期）
  - **When** 用户尝试恢复该发票
  - **Then** 后端返回错误提示"已超过 30 天恢复期限，无法恢复"，前端展示错误 Toast

- [ ] **AC-022**: 确认入库 — 日期为空时按钮置灰
  - **Given** Modal 中发票日期字段为空
  - **When** 用户查看"确认入库"按钮
  - **Then** "确认入库"按钮置灰不可点击，鼠标悬停时提示"请填写发票日期"

- [ ] **AC-023**: 确认入库 — 金额不合法时按钮置灰
  - **Given** Modal 中金额字段为空、为 0 或负数
  - **When** 用户查看"确认入库"按钮
  - **Then** "确认入库"按钮置灰不可点击，鼠标悬停时提示"金额必须大于 0"

- [ ] **AC-024**: 回收站入口角标更新
  - **Given** 当前有 3 张软删除发票
  - **When** 用户恢复其中 1 张
  - **Then** 回收站图标角标数字从 3 变为 2

- [ ] **AC-025**: 上传过程中按钮防重复
  - **Given** 用户已点击"上传"按钮，上传正在进行中
  - **When** 用户再次点击"上传"按钮
  - **Then** 按钮处于 loading 状态且不可重复点击

- [ ] **AC-026**: 列表加载中展示加载状态
  - **Given** 用户首次进入发票页面或切换 Tab
  - **When** 列表数据正在加载中
  - **Then** 展示骨架屏或 loading spinner，不展示空状态或旧数据

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-027**: 状态 Tab 与后端状态映射正确
  - **Given** 用户点击"待确认" Tab
  - **When** 前端发起 GET /api/invoices/ 请求
  - **Then** 请求参数 state 同时包含 pending 和 failed（或"待确认" Tab 发送两次请求分别获取两个状态并合并展示）

- [ ] **AC-028**: 动态字段展示取决于 invoice_type
  - **Given** Modal 中发票的 invoice_type = "高铁票"

  - **Given** 用户选择了 2025-01-01 到 2025-01-31 的日期范围
  - **When** 前端发送 GET /api/invoices/ 请求
  - **Then** 请求参数包含 date_from=2025-01-01 和 date_to=2025-01-31

- [ ] **AC-030**: 分页参数传递正确

  - **When** 前端发送请求
  - **Then** 请求参数为 page=3 和 page_size=50
