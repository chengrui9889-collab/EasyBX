# 替票管理 — 功能需求说明书

## 1. 背景与价值

**为什么做**：真实报销场景中，很多费用项目无法提供对应发票（如奖金、劳务费、无票支出），需要用其他发票作为"替票"来完成报销。当前系统缺少替票管理能力，用户无法在台账中表达"这笔费用是用那张发票替的"这一业务关系。

**做成什么样**：用户可在批次详情页中手动添加无发票的台账明细行，然后通过替票管理功能将已入库发票关联到这些行上，支持一对一替换、一对多拆分、多对一合并三种替票模式。关联后台账备注自动标注替票信息，替票行在表格中用特殊标记区分。替票发票按剩余金额可复用于多笔费用。

**关联**：`specs/开发路线图.md` Phase 2.1（替票后端）、Phase 2.2（替票前端）

---

## 2. 范围

### 本次做

- 新增台账明细行（手动添加无发票的费用行，字段：日期/事由/数量/单价/金额/垫款金额/备注）
- 替票管理独立入口（批次详情页的"替票管理"按钮）
- 三种替票模式：
  - **一对一替换**：一张替票发票替换一笔费用
  - **一对多拆分**：一张大额替票拆给多笔费用
  - **多对一合并**：多张小额替票合并覆盖一笔费用
- 替票关联列表展示 & 解除关联
- 台账替票标记（备注自动标注 + 特殊样式区分）
- 替票发票按剩余金额可复用于多笔费用

### 本次不做

- 替票金额与费用金额的差额自动处理
- 跨越批次的替票管理
- 台账明细行的批量导入
- 替票关联的审批流程

## 4. 需求详述

### 4.1 核心流程

```
批次详情页 → 台账表格底部 [+新增台账行]
  → 填写日期/事由/金额/数量/垫款金额/备注（金额可编辑，与有发票的行区分）
  → 台账表格中出现该行（无发票标记）

批次详情页 → [替票管理] 按钮 → 替票管理面板
  ├── 已有关联列表：展示所有已建立的替票关系
  │     ├── 被替换行的事由 + 金额
  │     ├── 替票发票号 + 金额
  │     ├── 替票模式
  │     └── [解除关联] 按钮
  │
  └── [新建替票关联] → 弹窗流程
        ├── 第1步：选择替票模式（一对一 / 一对多拆分 / 多对一合并）
        ├── 第2步：选择被替换的费用行（台账中已有的行，可选一行或多行）
        ├── 第3步：选择替票发票（从已入库且未被占用的发票库中筛选，按剩余金额可用）
        └── 第4步：确认 → 台账备注自动标注，替票发票标记为已占用
```

### 4.2 两种台账行类型

| 维度 | 有发票的行（现有） | 手动添加的行（本次新增） |
|------|-------------------|----------------------|
| 数据来源 | 发票加入批次时自动生成 | 用户手动添加 |
| 关联发票 | 有，一张发票对应一行 | 无，后续可通过替票关联发票 |
| 金额 | 来自发票，只读不可编辑 | 用户手动输入，可编辑 |
| 删除影响 | 发票回到可选池 | 不影响任何发票 |
| 替票关联 | 发票本身可作为替票 | 可作为被替换的费用行 |

### 4.3 手动台账行字段

| 字段 | 默认值 | 是否可编辑 | 规则 |
|------|--------|-----------|------|
| 日期 | 当天 | ✅ | — |
| 事由 | 空 | ✅ | 必填 |
| 数量 | 1 | ✅ | ≥ 1，允许小数 |
| 单价 | 自动计算 | ✅ | 单价 = 金额 ÷ 数量，修改数量后联动重算 |
| 金额 | 空 | ✅ | 必填，> 0 |
| 垫款金额 | 默认等于金额 | ✅ | 手动修改后不联动 |
| 备注 | 空 | ✅ | 手动填写，替票关联后自动追加替票信息 |

### 4.4 三种替票模式详述

#### 4.4.1 一对一替换

- **场景**：一笔无票费用（如奖金 1000 元），用一张金额匹配的发票替换
- **操作**：选择 1 个被替换行 → 选择 1 张替票发票 → 确认
- **检查**：替票发票金额 ≥ 被替换行金额
- **结果**：被替换行备注追加 `（替票发票号）`

#### 4.4.2 一对多拆分

- **场景**：一张大额咨询发票（3000 元），拆给三笔劳务费（各 1000 元）
- **操作**：选择 1 张替票发票 → 选择 N 个被替换行 → 确认
- **检查**：替票发票金额 ≥ 所有被替换行金额之和
- **结果**：每个被替换行备注均追加替票信息

#### 4.4.3 多对一合并

- **场景**：三张小额餐饮发票（合计 1000 元），合并覆盖一笔团建费（1000 元）
- **操作**：选择 N 张替票发票 → 选择 1 个被替换行 → 确认
- **检查**：所有替票发票金额之和 ≥ 被替换行金额
- **结果**：被替换行备注追加多张替票信息，格式 `费用事由（替票1011, 替票1012, 替票1013）`

### 4.5 替票发票可用规则

- **来源**：当前批次所在的批次中，状态为 `confirmed`（已入库）且未被占用的发票
- **占用规则**：替票关联确认后，替票发票标记为已占用（与普通发票加入批次效果一致）
- **按剩余金额可用**：一张替票发票可被用于多个替票关系，各被替换行的金额之和不超过该发票的总金额
- **已被占用的发票**：不可选为替票（包括已被其他批次占用和已被当前批次其他替票关系占用的发票）

### 4.6 替票关联的生命周期

| 操作 | 替票关联处理 | 替票发票状态变化 |
|------|------------|----------------|
| 解除替票关联 | 关联删除 | 发票回到可选池（解除占用） |
| 删除被替换的台账行 | 关联自动解除 | 发票回到可选池 |
| 删除批次 | 所有关联自动解除 | 所有替票发票回到可选池 |
| 导出台账 | 不受影响 | 不变 |

### 4.7 台账备注替票格式

- **单张替票**：`国际商务微课制作奖金（替票1011）`
- **多张替票**：`团建费（替票1011, 替票1012, 替票1013）`
- **追加规则**：若该行已有手动备注，替票信息追加到末尾

### 4.8 数据隔离

- 替票关联仅在同一批次内有效
- 手动添加的台账行不影响任何发票数据
- 替票关联不影响发票的原始数据（金额、日期等不变）

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001：新增台账明细行**
  - **Given** 用户在批次详情页的台账表格区域
  - **When** 点击表格底部的"+新增台账行"
  - **Then** 表格底部新增一行空白行，日期/事由/金额/数量/垫款金额/备注均为可编辑状态

- [ ] **AC-002：填写台账明细行并保存**
  - **Given** 新增了一行空白台账行
  - **When** 用户填写日期="2025-12-01"、事由="奖金"、金额=1000、数量=1、垫款金额=1000
  - **Then** 该行数据保存到批次，单价自动计算为 1000.00

- [ ] **AC-003：手动台账行正常导出台账**
  - **Given** 批次中有一条手动添加的台账行（金额=1000）
  - **When** 导出台账 Excel
  - **Then** 该行正常出现在台账中，与其他有发票的行格式一致

- [ ] **AC-004：替票管理入口**
  - **Given** 用户在批次详情页
  - **When** 点击"替票管理"按钮
  - **Then** 弹出替票管理面板，展示"已有关联"列表和"新建替票关联"入口

- [ ] **AC-005：一对一替票关联**
  - **Given** 台账中有一条手动添加的"奖金"行（金额=1000），发票库中有一张金额=1000 的交通发票
  - **When** 选择替票模式=一对一 → 选择"奖金"行为被替换行 → 选择交通发票为替票 → 确认
  - **Then** 关联成功，"奖金"行的备注更新为"奖金（替票发票号）"，替票发票标记为已占用

- [ ] **AC-006：一对多拆分替票关联**
  - **Given** 一张咨询发票金额=3000，台账中有 3 条劳务费行（各 1000）
  - **When** 选择替票模式=一对多拆分 → 选择咨询发票为替票 → 勾选 3 条劳务费行为被替换行 → 确认
  - **Then** 3 条劳务费行的备注均标注该咨询发票的替票信息，咨询发票标记为已占用

- [ ] **AC-007：多对一合并替票关联**
  - **Given** 3 张餐饮发票（合计=1000），台账中有 1 条"团建费"行（金额=1000）
  - **When** 选择替票模式=多对一合并 → 勾选 3 张餐饮发票为替票 → 选择"团建费"行为被替换行 → 确认
  - **Then** "团建费"行备注标注"团建费（替票1011, 替票1012, 替票1013）"，3 张发票标记为已占用

- [ ] **AC-008：替票关联列表展示**
  - **Given** 批次已有多个替票关联
  - **When** 打开替票管理面板
  - **Then** 列表中每条展示：被替换行的事由+金额、替票发票号+金额、替票模式，并提供"解除关联"操作按钮

- [ ] **AC-009：替票关联的台账特殊标记**
  - **Given** 台账中有替票关联的行
  - **When** 查看台账表格
  - **Then** 该行备注包含替票信息（格式如"奖金（替票1011）"），并且该行有特殊样式标记（如浅蓝色背景或替票图标）

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-010：替票金额不足时不可确认**
  - **Given** 替票发票金额=500，被替换行金额=1000
  - **When** 在一对一模式下选择该发票并点击确认
  - **Then** 提示"替票发票金额（500.00）不足以替换该费用行（1000.00）"，确认按钮置灰不可操作

- [ ] **AC-011：替票发票已被占用不可选**
  - **Given** 发票 A 已被其他批次占用
  - **When** 在替票选择器中搜索发票 A
  - **Then** 发票 A 不在可选列表中

- [ ] **AC-012：替票发票剩余金额不足时显示提示**
  - **Given** 发票 A（金额=3000）已替了 2000 元的费用行（剩余 1000），台账中有一行 2000 元的费用行
  - **When** 在一对一模式下选择发票 A 并确认
  - **Then** 提示"该发票剩余金额（1000.00）不足以替换该费用行（2000.00）"

- [ ] **AC-013：替票发票按剩余金额可供多笔拆分**
  - **Given** 发票 A（金额=3000）已替了 1000 元的费用（剩余 2000），台账中还有一行 1500 元的费用行
  - **When** 在一对一模式下选择发票 A 并确认
  - **Then** 关联成功，剩余金额更新为 500

- [ ] **AC-014：解除替票关联 → 二次确认**
  - **Given** 替票管理面板中有一条已有关联
  - **When** 用户点击"解除关联"
  - **Then** 弹出确认提示"确定解除该替票关联吗？解除后替票发票将回到可选池"，确认后关联解除，发票回到可选池

- [ ] **AC-015：删除有替票关联的台账行 → 自动解除关联**
  - **Given** 某台账行有关联的替票
  - **When** 用户删除该台账行
  - **Then** 替票关联自动解除，替票发票回到可选池

- [ ] **AC-016：新增台账行金额不能为零**
  - **Given** 用户在新增台账行
  - **When** 金额字段输入 0 或不填，点击保存
  - **Then** 提示"金额不能为 0 或空"，保存被阻止

- [ ] **AC-017：替票选择器搜索过滤**
  - **Given** 替票选择器打开，发票库中有多张发票
  - **When** 用户在搜索框中输入关键词
  - **Then** 列表过滤出匹配的发票（按发票号/销售方/金额过滤）

- [ ] **AC-018：无可选替票发票时的空状态**
  - **Given** 所有已入库发票均被占用
  - **When** 打开替票选择器
  - **Then** 显示"暂无可选替票发票"空状态提示

### 5.3 业务规则 (Business Rules)

- [ ] **AC-019：替票发票来源规则** ← BR-001
  - **Given** 发票库中有 confirmed、processing、pending 三种状态的发票
  - **When** 打开替票选择器
  - **Then** 仅展示已入库（confirmed）且未被占用的发票

- [ ] **AC-020：金额差额不自动处理** ← BR-002
  - **Given** 替票金额=800，费用金额=1000
  - **When** 替票关联确认后
  - **Then** 系统不做任何金额调整或差额提示，台账中金额保持 1000

- [ ] **AC-021：替票发票按剩余金额可用** ← BR-003
  - **Given** 发票 A（金额=3000）已替了 1000 元的费用行
  - **When** 再次选择发票 A 作为替票
  - **Then** 发票 A 仍可选，剩余可替金额为 2000

- [ ] **AC-022：台账备注替票格式** ← BR-004
  - **Given** 某台账行被替票关联
  - **When** 关联确认后查看台账
  - **Then** 备注格式为「费用事由（替票发票号）」，多张替票用逗号分隔：「费用事由（替票1011, 替票1012）」

- [ ] **AC-023：新增台账行删除不影响发票** ← BR-005
  - **Given** 一条手动添加的台账行（无替票关联）
  - **When** 用户删除该行
  - **Then** 该行被删除，不影响任何发票的状态

- [ ] **AC-024：替票关联确认后发票标记为已占用** ← BR-006
  - **Given** 发票 A 被选为替票并确认关联
  - **When** 在其他批次的发票选择器或替票选择器中搜索发票 A
  - **Then** 发票 A 不在可选列表中

- [ ] **AC-025：替票仅在同一批次内有效** ← BR-007
  - **Given** 批次 X 中有替票关联
  - **When** 在批次 Y 中查看任何数据
  - **Then** 批次 X 的替票关联不影响批次 Y 的任何操作

- [ ] **AC-026：手动台账行可编辑** ← BR-008
  - **Given** 一条手动添加的台账行（金额=1000）
  - **When** 用户将金额改为 1200
  - **Then** 保存成功，金额变为 1200，单价联动重算
