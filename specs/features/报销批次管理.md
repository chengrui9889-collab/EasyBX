# 报销批次 + 台账导出 — 功能需求说明书

## 1. 背景与价值

**为什么做**：用户完成发票入库后，下一步是将多张发票组织成一个报销批次，并按公司台账模版生成 Excel 台账提交财务。当前缺少「批次管理」这一核心业务环节，发票入库后无法串联为可报销的结构化台账。

**做成什么样**：用户创建报销批次（自动填充账号默认值），从已入库发票中搜索勾选批量加入，在台账预览表格中逐行调整数量/单价/垫款金额/备注，一键导出符合公司模版的 Excel 台账文件。整个过程所见即所得，修改只影响台账输出，不动原始发票。

**关联**：`specs/开发路线图.md` Phase 1.5（批次后端）、Phase 1.6（台账 Excel）、Phase 1.7（批次前端）

## 2. 范围

### 本次做

- 批次 CRUD（创建、查看列表、查看详情、编辑、删除）
- 创建批次时自动填充账号默认值（报账部门、报账人、收款人、银行卡号、开户行）
- 台账预览表格（日期/事由/数量/单价/金额/垫款金额/备注，支持行内编辑）
- 金额不可编辑（来自发票原始数据）
- 数量 × 单价 = 金额 自动联动（数量允许小数，≥ 1）
- 垫款金额默认等于金额，用户可手动修改
- 备注自动填充（出行信息拼接 / 发票已有备注）
- 发票搜索选择器（关键词搜索 + 查看发票原文件 + 多选批量加入）
- 从批次移除发票（回到可选池）
- 导出 Excel 台账（基于 `文件/台账模版.xlsx` 模版填充）

### 本次不做

- 替票关联（Phase 2.1）
- PDF 导出（Phase 2.3）
- 费用报销单预览（Phase 2.5）
- 发票归档（Phase 3.1）
- 批次审批流
- 台账模版自定义

## 4. 需求详述

### 4.1 核心流程

```
用户登录 → 进入批次列表页 → 创建批次（自动填默认值 + 手动填期间+报账日期）
  → 进入批次详情页 → 打开发票选择器（搜索 + 预览 + 批量勾选）
  → 发票加入批次 → 台账预览表格展示每一行数据
  → 用户编辑数量/单价/垫款金额/备注 → 金额不可改
  → 导出台账 Excel（基于模版填充，含合计行）
```

### 4.2 批次生命周期

- 批次创建后可反复编辑（加减发票、修改台账字段）
- 导出台账后批次不锁定，可继续修改和再次导出
- 删除批次后，关联发票全部回到可选池

### 4.3 创建批次字段

| 字段 | 默认值来源 | 是否必填 | 能否修改 |
|------|-----------|---------|---------|
| 报账部门 | 账号默认值 | 否 | 可修改 |
| 报账期间 | 无，手动填写 | 否 | 可修改 |
| 报账人 | 账号默认值 | 否 | 可修改 |
| 收款人 | 账号默认值 | 否 | 可修改 |
| 银行卡号 | 账号默认值 | 否 | 可修改 |
| 开户行 | 账号默认值 | 否 | 可修改 |
| 报账日期 | 当天系统日期 | 否 | 可修改 |
| 审核人 | 留空 | 否 | 可修改 |

### 4.4 台账表格列定义

| 列名 | 数据来源 | 是否可编辑 | 联动规则 |
|------|---------|-----------|---------|
| 日期 | 发票的 `invoice_date` | ❌ | — |
| 事由 | 发票的 `category`（报销类型） | ❌ | — |
| 数量 | 默认 = 1 | ✅ | 修改后 → 单价重算 |
| 单价 | 默认 = 金额 ÷ 1 | ✅ | 修改后 → 金额不变 |
| 金额 | 发票的 `amount` | ❌（只读） | — |
| 垫款金额 | 默认 = 金额 | ✅ | 手动修改后不联动 |
| 备注 | 自动填充 | ✅ | 见 4.4.1 |

#### 4.4.1 备注自动填充规则

1. 若发票已有 `remark` → 使用 remark
2. 若无 remark 但有出行信息：
   - 高铁票：`{departure_station}→{arrival_station}`
   - 滴滴打车：`{departure_location}→{arrival_location}`
   - 飞机行程单：`{departure_city}→{arrival_city}`
3. 若两者均无 → 备注留空

### 4.5 台账 Excel 导出规则

- **模版文件**：`文件/台账模版.xlsx`（Sheet1）
- **标题**：保持模版标题「图联科技报销台账」
- **表头**：A2 单元格填充 `报账部门：{部门}  报账期间：{期间}`
- **数据区**：从第 4 行开始逐行填充每张发票的数据（日期/事由/数量/单价/金额/垫款金额/备注）
- **合计行**：紧跟最后一行数据，F 列（垫款金额）写入 SUM 公式
- **底部**：A21 填充审核人/报账人/报账日期/合计金额，A22 填充收款人/银行卡号/开户行
- **文件名**：`{报账部门}_{报账期间}_台账.xlsx`

### 4.6 发票选择器

- **可选发票条件**：状态 = `confirmed`（已入库）且未被任何批次占用
- **展示信息**：发票类型图标、金额、日期、报销类型、销售方名称
- **搜索**：支持按发票号码、销售方名称、报销类型进行关键词过滤
- **预览**：点击发票可预览原始文件（图片直接渲染，PDF 用 iframe）
- **选择方式**：多选勾选，点击"确认"后批量加入批次

### 4.7 数据隔离

- 一张发票只能属于一个批次（加入批次 X 后不再出现在任何选择器中）
- 从批次移除发票 → 回到可选池，可被其他批次选用
- 删除批次 → 所有关联发票回到可选池
- 批次内的修改（数量、单价、垫款金额、备注）**不影响原始发票数据**

### 4.8 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 只有 `confirmed`（已入库）状态的发票可加入批次 |
| BR-002 | 一张发票只能属于一个批次 |
| BR-003 | 金额不可编辑，来自发票原始数据 |
| BR-004 | 数量 ≥ 1，允许小数 |
| BR-005 | 单价 = 金额 ÷ 数量（精确到两位小数） |
| BR-006 | 垫款金额默认 = 金额，用户可手动修改 |
| BR-007 | 台账合计行对垫款金额列求和 |
| BR-008 | 删除批次后所有关联发票回到可选池 |
| BR-009 | 批次导出后不锁定，可继续编辑和再次导出 |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001：创建批次**
  - **Given** 用户已登录且在批次列表页
  - **When** 点击"新建批次"，报账部门/报账人/收款人/银行卡号/开户行自动填充账号默认值，报账期间手动填写，报账日期默认当天、审核人留空，点击保存
  - **Then** 批次创建成功，出现在批次列表页

- [ ] **AC-002：批次列表展示**
  - **Given** 用户已登录且有多个批次
  - **When** 进入批次列表页
  - **Then** 每个批次卡片展示：报账部门、报账期间、发票数量、创建日期，按创建日期倒序排列

- [ ] **AC-003：台账预览表格**
  - **Given** 批次已关联了发票
  - **When** 用户进入批次详情页
  - **Then** 展示台账预览表格：日期/事由/数量/单价/金额/垫款金额/备注，每行对应一张发票，初始值按 4.4 节规则填充

- [ ] **AC-004：备注自动填充**
  - **Given** 发票有出行信息或已有 remark
  - **When** 发票加入批次后显示在台账表格中
  - **Then** 备注按 4.4.1 节规则自动填充

- [ ] **AC-005：修改数量 → 单价联动**
  - **Given** 台账表格中某行金额为 100，数量为 1，单价为 100
  - **When** 用户将数量改为 4
  - **Then** 金额保持 100，单价自动更新为 25.00

- [ ] **AC-006：修改垫款金额**
  - **Given** 某行金额为 100，垫款金额为 100
  - **When** 用户将垫款金额改为 80
  - **Then** 垫款金额变为 80，金额和单价不变；后续修改数量时垫款金额保持 80 不变

- [ ] **AC-007：修改备注**
  - **Given** 台账表格某行备注为"合肥→淮南南"
  - **When** 用户修改备注为"出差交通费"
  - **Then** 备注更新为"出差交通费"

- [ ] **AC-008：搜索并批量加入发票**
  - **Given** 用户在批次详情页，已入库发票池中有多张未被占用的发票
  - **When** 点击"添加发票"，在搜索框中输入关键词，勾选多张发票后点击确认
  - **Then** 已选发票批量加入批次，台账表格新增对应行

- [ ] **AC-009：选择器中预览发票原文件**
  - **Given** 用户在发票选择器中
  - **When** 点击某张发票的"预览"按钮
  - **Then** 展示该发票的原始文件（图片直接渲染，PDF 用 iframe）

- [ ] **AC-010：从批次移除发票**
  - **Given** 批次台账表格中有一张发票
  - **When** 用户点击该行的"移除"按钮并确认
  - **Then** 发票从批次中移除，回到可选池，台账表格中该行消失

- [ ] **AC-011：导出台账 Excel**
  - **Given** 批次已关联发票
  - **When** 用户点击"导出台账"
  - **Then** 下载 Excel 文件，内容按 4.5 节规则填充模版，文件名格式为 `{报账部门}_{报账期间}_台账.xlsx`

- [ ] **AC-012：创建批次时默认值填充**
  - **Given** 用户在账号设置中已配置默认的报账部门为"产教融合"、报账人为"程瑞"、收款人为"程瑞"
  - **When** 用户点击新建批次
  - **Then** 报账部门自动填充"产教融合"、报账人自动填充"程瑞"、收款人自动填充"程瑞"、报账日期默认当天、审核人留空、报账期间为空白

- [ ] **AC-013：导出后批次仍可编辑**
  - **Given** 批次已导出台账
  - **When** 用户再次进入批次详情页修改数量并再次导出
  - **Then** 修改和导出均正常工作

- [ ] **AC-031：查看账号默认值**
  - **Given** 用户在"账号设置"页
  - **When** 进入页面
  - **Then** 展示当前账号的默认报账部门、报账人、收款人、银行卡号、开户行，各字段可编辑

- [ ] **AC-032：编辑账号默认值**
  - **Given** 用户在"账号设置"页，当前部门为空
  - **When** 将默认部门修改为"产教融合"，点击保存
  - **Then** 保存成功，新建批次时部门自动填充"产教融合"

- [ ] **AC-033：添加发票时预览原始发票**
  - **Given** 用户在添加发票弹窗中
  - **When** 点击某张发票行的"预览"按钮
  - **Then** 弹出模态框展示该发票的原始文件（图片或 PDF）

- [ ] **AC-014：删除批次**
  - **Given** 批次关联了 5 张发票
  - **When** 用户删除该批次并确认
  - **Then** 批次及关联关系被删除，5 张发票回到可选池

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-015：移除发票 → 二次确认**
  - **Given** 批次台账表格中有一张发票
  - **When** 用户点击"移除"
  - **Then** 弹出确认提示"确定移除该发票吗？移除后发票可被其他批次选用"，确认后才执行移除

- [ ] **AC-016：删除批次 → 二次确认**
  - **Given** 批次详情页
  - **When** 用户点击"删除批次"
  - **Then** 弹出确认提示"确定删除该批次吗？批次内发票将回到可选池"，确认后才执行删除

- [ ] **AC-017：空批次不允许导出**
  - **Given** 批次未关联任何发票
  - **When** 用户点击"导出台账"
  - **Then** 按钮置灰不可点击，或点击后提示"请先添加发票"

- [ ] **AC-018：所有发票已被占用**
  - **Given** 所有已入库发票都已属于其他批次
  - **When** 用户打开发票选择器
  - **Then** 列表为空，展示"暂无可选发票，所有已入库发票已被其他批次占用"

- [ ] **AC-019：已在本批次的发票不出现在选择器中**
  - **Given** 发票 A 已在当前批次中
  - **When** 用户打开发票选择器
  - **Then** 发票 A 不在可选列表中

- [ ] **AC-020：数量小于 1 → 恢复为 1**
  - **Given** 台账表格中某行数量为 1
  - **When** 用户将数量改为 0 或 -1，然后失焦
  - **Then** 数量恢复为 1，行内显示红色提示"数量不能小于 1"

- [ ] **AC-021：批次列表为空**
  - **Given** 用户从未创建过任何批次
  - **When** 进入批次列表页
  - **Then** 展示空状态引导："暂无报销批次，点击新建开始创建"

- [ ] **AC-022：金额为 0 时的台账行处理**
  - **Given** 某发票金额为 0
  - **When** 该发票计入台账
  - **Then** 金额列显示 0.00，数量/单价/垫款金额正常渲染，导出时该行正常写入

- [ ] **AC-023：批量加入较多发票**
  - **Given** 可选发票超过 50 张
  - **When** 用户勾选 15 张发票并确认加入
  - **Then** 台账表格正确渲染所有 15 行，页面无卡顿

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-024：一张发票只能属于一个批次** ← BR-002
  - **Given** 发票 A 已加入批次 X
  - **When** 用户打开批次 Y 的发票选择器
  - **Then** 发票 A 不在可选列表中

- [ ] **AC-025：只有已入库发票可加入批次** ← BR-001
  - **Given** 有处理中、待确认、已入库三个状态的发票
  - **When** 用户打开发票选择器
  - **Then** 仅展示已入库（confirmed）状态的发票

- [ ] **AC-026：批次修改不影响原始发票** ← BR-003, BR-006
  - **Given** 发票 A（金额=34.00，备注"合肥→淮南南"）加入批次后，用户在台账中将备注改为"出差车费"，垫款金额改为 30
  - **When** 用户去发票管理页查看发票 A
  - **Then** 发票 A 的金额仍为 34.00，备注仍为"合肥→淮南南"，未受批次修改影响

- [ ] **AC-027：单价 = 金额 ÷ 数量** ← BR-005
  - **Given** 台账中某行金额 = 37.00，数量 = 2
  - **When** 该行渲染
  - **Then** 单价显示为 18.50

- [ ] **AC-028：垫款金额默认 = 金额** ← BR-006
  - **Given** 发票加入批次，金额 = 34.00
  - **When** 台账表格渲染该行
  - **Then** 垫款金额初始值为 34.00

- [ ] **AC-029：合计行对垫款金额求和** ← BR-007
  - **Given** 台账有 3 行数据，垫款金额分别为 100、200、300
  - **When** 导出台账
  - **Then** 合计行垫款金额 = 600，金额列不做合计

- [ ] **AC-030：删除批次后发票恢复可选** ← BR-008
  - **Given** 发票 A 在批次 X 中且批次 X 被删除
  - **When** 用户打开任意批次的选择器
  - **Then** 发票 A 出现在可选列表中
