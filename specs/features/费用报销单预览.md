# 费用报销单预览 — 功能需求说明书

## 1. 背景与价值

**为什么做**：用户在完成报销批次的发票勾选和台账编辑后，需要一张符合传统纸质格式的"费用报销单"来提交财务。台账是明细表，报销单是按事由合并的汇总表，两种格式各有用途。当前缺少报销单预览功能，用户需要手动把台账数据誊写到纸质报销单上。

**做成什么样**：用户从批次详情页一键进入 A4 纸风格的报销单预览页，系统自动按发票类别合并数据、拆分金额到传统格位、生成大写合计金额，严格还原纸质费用报销单的所有视觉元素（标题双下划线、表头线框、表格框线、签名区等），打印后用户可手写补充签名和备注。

**关联**：`specs/开发路线图.md` Phase 2.5（后端）、Phase 2.6（前端）

## 2. 范围

### 本次做

- A4 纸风格报销单预览页面（左侧边栏 + 顶部工具栏 + 中间预览区）
- 从批次详情页点击按钮跳转进入
- 按发票类别合并展示（同类别金额求和为一行）
- 金额拆分到传统格位（百十万千百十元角分）
- 合计金额转中文大写（金额(大写)：X万X仟X佰X拾X元X角X分）
- 固定 6 行数据区，类别超过 6 个自动续页
- 表头（报销部门 + 年月日）和报销人自动从批次信息填充
- 严格还原纸质费用报销单的视觉细节（框线、双下划线标题、格位分割）

### 本次不做

- 打印功能 — 后续迭代
- 导出 PDF 功能 — 后续迭代（Phase 2.5）
- 摘要/备注列自动填充 — 留空供用户打印后手写
- "单据及附件共 X 页"自动计算 — 留空供用户手写
- 原借款/应退补款 — 留空
- 领导审批/会计主管/复核/出纳签名区 — 留空
- 替票信息在报销单中标注 — 后续迭代考虑

## 4. 需求详述

### 4.1 核心流程

1. 用户在批次详情页，点击"预览报销单"按钮
2. 系统跳转到报销单预览页，以 A4 纸居中展示
3. 系统读取批次信息，填充表头（报销部门 + 年月日）
4. 系统按发票类别分组，计算同类金额总和
5. 数据行渲染：每行展示类别名称 + 金额拆分到格位
6. 若类别数 > 6，超出部分续到第二页
7. 底部渲染：合计行 + 金额大写 + 签名区（报销人自动填充，其余留空）

### 4.3 续页规则

- 每页固定 6 行数据区
- 第 1 页展示前 6 个类别，第 2 页展示第 7-12 个类别，以此类推
- 每页都有独立的表头（报销部门 + 年月日）、表格（6 行数据区）、合计行和底部签名区
- 每页的合计金额为该页数据之和

### 4.4 金额格位填充规则

- 从最高位（百万）到最低位（角分），将金额数字逐位填入对应格位
- 例：金额 `168.00` → 百万格空、十万格空、万格空、千格空、百格 `1`、十格 `6`、个格 `8`、角格 `0`、分格 `0`
- 例：金额 `1,234,567.89` → 百万格 `1`、十万格 `2`、万格 `3`、千格 `4`、百格 `5`、十格 `6`、个格 `7`、角格 `8`、分格 `9`
- 金额为 `0.00` 时所有格位为空

### 4.5 金额大写规则

- 合计金额转换为中文大写数字 + 单位
- 例：`468.00` → "肆佰陆拾捌元整"
- 例：`0.00` → "零元整"
- 例：`100.50` → "壹佰元伍角整"
- 只处理"元/角/分"，不做"万/仟/佰/拾"的额外层级拆分

### 4.6 交互规则

- 批次详情页"预览报销单"按钮：批次有发票时正常可点，无发票时置灰
- 报销单预览页顶部工具栏提供"返回"按钮回到批次详情页
- 预览页内容以 A4 纸比例居中展示在中间区域
- 所有留空字段（摘要、备注、签名区等）以空白单元格呈现，不显示占位符

### 4.7 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 合并依据为发票的 `category`（报销类型/类别），同一类别的所有发票合并为一行 |
| BR-002 | 合行金额 = 该类别所有发票金额之和 |
| BR-003 | 金额列按传统格位拆分展示：百、十、万、千、百、十、元（个位） |
| BR-004 | 报销单上的修改不影响原始发票数据和批次台账数据 |
| BR-005 | 每页固定 6 行数据区，超出自动续页 |
| BR-006 | 「摘要」列始终空白 |
| BR-007 | 「备注」列始终空白 |
| BR-008 | 「单据及附件共＿页」始终空白 |
| BR-009 | 「原借款」「应退(补)款」始终空白 |
| BR-010 | 「领导审批」「会计主管」「复核」「出纳」始终空白 |
| BR-011 | 「报销人」自动填充批次的报账人姓名 |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001：从批次详情进入预览**
  - **Given** 用户已登录且在批次详情页，该批次已关联发票
  - **When** 用户点击"预览报销单"按钮
  - **Then** 跳转到报销单预览页，页面以 A4 纸风格居中展示费用报销单

- [ ] **AC-002：表头信息自动填充**
  - **Given** 批次的报账部门为"产教融合"、报账日期为"2025年12月20日"
  - **When** 报销单预览渲染完成
  - **Then** 「报销部门：」处显示"产教融合"，「年 月 日」处显示"2025 年 12 月 20 日"

- [ ] **AC-003：按事由合并展示**
  - **Given** 批次包含 3 张"交通费"（金额 34、56、78）和 2 张"餐饮费"（金额 100、200）
  - **When** 报销单表格渲染完成
  - **Then** 第 1 行数据：报销项目为"交通费"；第 2 行数据：报销项目为"餐饮费"；第 3-6 行数据行为空行

- [ ] **AC-004：金额拆分到各格位**
  - **Given** "交通费"行合计金额为 168.00
  - **When** 该行金额格位渲染
  - **Then** 百位格显示 `1`，十位格显示 `6`，个位格（元）显示 `8`，角格 `0`，分格 `0`，更高位格（千/万/十万/百万）为空

- [ ] **AC-005：空白列保持留空**
  - **Given** 报销单表格任意数据行
  - **When** 表格渲染完成
  - **Then** 「摘要」列所有格均为空白，「备注」列所有格均为空白

- [ ] **AC-006：合计行正确计算**
  - **Given** 表格中有两行数据，金额分别为 168.00 和 300.00
  - **When** 合计行渲染
  - **Then** 合计行金额各格位正确展示 468.00 的拆分结果

- [ ] **AC-007：金额大写自动生成**
  - **Given** 合计金额为 468.00
  - **When** 底部金额大写区域渲染
  - **Then** 显示"肆佰陆拾捌元整"

- [ ] **AC-008：底部签名区 — 报销人自动填充**
  - **Given** 批次的报账人为"程瑞"
  - **When** 底部签名栏渲染
  - **Then** 「报销人：」处显示"程瑞"，「领导审批」、「会计主管」、「复核」、「出纳」处为空白

- [ ] **AC-009：底部其他字段留空**
  - **Given** 报销单完整渲染
  - **When** 查看表头和底部各字段
  - **Then** 「单据及附件共＿页」页数处留空，「原借款」、「应退(补)款」均留空

- [ ] **AC-010：续页处理 — 类别超过 6 个**
  - **Given** 批次包含 8 种不同的发票类别
  - **When** 报销单渲染
  - **Then** 第 1 页显示前 6 个类别 + 合计行 + 完整表头表尾，第 2 页显示剩余 2 个类别 + 合计行 + 完整表头表尾；其余 4 行数据行为空行

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-011：仅有一个类别**
  - **Given** 批次内所有发票都属于"交通费"类别，总金额 500.00
  - **When** 报销单渲染
  - **Then** 表格第 1 行显示"交通费"及 500.00 拆分结果，第 2-6 行数据行为空行，合计行正常显示

- [ ] **AC-012：金额为零**
  - **Given** 某类别发票金额合计为 0
  - **When** 该行金额格位渲染
  - **Then** 所有金额格位均为空

- [ ] **AC-013：大额金额溢出**
  - **Given** 某类别金额合计为 1,234,567.89
  - **When** 该行金额格位渲染
  - **Then** 百万位格显示 `1`，十万位格 `2`，万位格 `3`，千位格 `4`，百位格 `5`，十位格 `6`，个位格 `7`，角格 `8`，分格 `9`，表格宽度足以容纳

- [ ] **AC-014：空批次不允许预览**
  - **Given** 批次未关联任何发票
  - **When** 用户查看批次详情页
  - **Then** "预览报销单"按钮置灰不可点击

- [ ] **AC-015：类别超过一页时各页合计独立**
  - **Given** 批次有 8 个类别，前 6 个金额和为 1000，后 2 个金额和为 500
  - **When** 报销单渲染
  - **Then** 第 1 页合计行为 1000，第 2 页合计行为 500

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-016：合并依据为发票类别** ← BR-001
  - **Given** 批次有 3 张发票，category 分别为 "交通费"、"交通费"、"餐饮费"
  - **When** 报销单生成
  - **Then** 产生 2 行数据行：第 1 行"交通费"金额为两张交通费发票之和，第 2 行"餐饮费"金额为该餐饮费发票金额

- [ ] **AC-017：金额 = 同类发票金额求和** ← BR-002
  - **Given** "交通费"下有两张发票，金额分别为 34.50 和 56.50
  - **When** 报销单该行渲染
  - **Then** 金额格位显示 91.00 的拆分结果（十位格 `9`，个位格 `1`，角格 `0`，分格 `0`）

- [ ] **AC-018：摘要/备注始终留空** ← BR-006, BR-007
  - **Given** 任何有数据的报销单行
  - **When** 报销单渲染
  - **Then** 所有数据行的「摘要」、「备注」列均为空白

- [ ] **AC-019：固定字段始终留空** ← BR-008, BR-009, BR-010
  - **Given** 报销单完整渲染
  - **When** 查看表头"单据及附件共＿页"、底部"原借款"、"应退(补)款"、"领导审批"、"会计主管"、"复核"、"出纳"
  - **Then** 以上所有位置均为空白

- [ ] **AC-020：报销人不影响原始批次数据** ← BR-004
  - **Given** 报销单页面渲染完成
  - **When** 用户返回批次详情页查看台账
  - **Then** 台账数据（数量、单价、金额、垫款金额、备注）与进入预览前完全一致
