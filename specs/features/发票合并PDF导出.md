# 发票合并 PDF 导出 — 功能需求说明书

## 1. 背景与价值

**为什么做**：用户完成报销批次整理后，需要将批次内所有发票合并为一个 PDF 文件，打印出来连同费用报销单一并提交给财务。纸质发票整理和打印是报销流程的最后一步，目前缺少"一键导出合并 PDF"的能力，用户需要逐张打开发票再手动打印，效率低下。

**做成什么样**：用户在批次详情页或发票列表页选择要导出的发票，系统自动按发票类型分组展示，用户可为每组分别选择排版方式（A4 纵向/横向），最终生成一个可直接打印的合并 PDF 文件供下载。

**关联**：`specs/开发路线图.md` Phase 2.3 + Phase 2.4

---

## 2. 范围

### 本次做

- 后端生成合并 PDF：按发票类型分组排版，同类型发票放在同一页或连续页
- 每种发票类型可独立选择排版模式：A4 纵向（一页 2 张，上下排列）或 A4 横向（一页 4 张，2×2 排列）
- 发票原样输出：不加页眉、页码、发票号码标注等任何辅助信息
- 支持两种导出来源：
  - 按批次导出：导出批次关联的所有发票
  - 自由勾选导出：从发票列表页勾选指定发票导出
- PDF/图片统一处理：JPG/PNG 按比例缩放塞入格子，PDF 提取页面渲染
- 前端排版选择弹窗：按发票类型分组展示，每项可单独选择排版 + 显示该类型发票数量
- 批次详情页和发票列表页的"导出 PDF"入口

### 本次不做

- 费用报销单合并（Phase 2.5-2.6 单独处理）
- 页码、页眉、水印等辅助信息（保持发票原始面貌）
- 发票内容裁剪/优化（原样输出，不做二次处理）
- 批量导出多个批次合并为一个 PDF
- 在线 PDF 预览（直接用浏览器下载后打开）

## 4. 需求详述

### 4.1 核心流程

```
用户在批次详情页（或发票列表页勾选后）
  │
  ├── 点击"导出 PDF"按钮
  │     │
  │     └── 弹出排版选择弹窗
  │           ├── 按 invoice_type 分组展示所有发票类型
  │           ├── 每组显示：类型名称 + 数量 + 排版下拉选择
  │           ├── 支持"统一设置"快捷操作（一键设为全部纵向/全部横向）
  │           └── 点击"确认导出"
  │
  └── 后端生成合并 PDF
        │
        ├── 按 invoice_type（发票类型）分组
        ├── 同类型发票连续排放，不同类型不混排在同一页
        ├── 每组按各自选定的排版填充每页
        └── 返回 PDF 文件下载
```

### 4.2 排版规则细则

**A4 纵向（portrait）**：
- 一页放置 2 张发票，上下排列
- 上半页和下半页各占约 50% 高度
- 发票按比例缩放，居中显示

**A4 横向（landscape）**：
- 一页放置 4 张发票，2×2 排列
- 每个格子占约 25% 面积
- 发票按比例缩放，居中显示

**分组排放规则**：
- 按 `invoice_type` 分组，同类型发票连续排放
- 不同类型发票不混排在同一页
- 如果某一组发票数量不足以填满一页（如纵向模式下只有 1 张），则该页只放实际数量的发票，剩余位置留空
- 组与组之间直接衔接，不插入额外空白页

**文件处理规则**：
- PDF 发票：提取 PDF 页面内容，按格子尺寸缩放到 A4 页面对应位置
- JPG/PNG 图片：按原始比例缩放，适配到格子内居中显示
- 不添加任何边框、标注、页眉、页码等辅助元素

### 4.3 交互规则

- 导出弹窗按发票类型分组展示，每组显示类型名称、发票数量、排版选择器
- 提供"统一设为纵向"和"统一设为横向"快捷操作，避免逐项修改
- 每种类型的排版选择器默认为纵向
- 确认导出后，按钮变为加载状态，防止重复提交
- 导出完成后自动触发文件下载
- 如果选中发票为空，"导出 PDF"按钮置灰不可点击

### 4.4 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 按 `invoice_type` 分组排放，同类型连续，不同类型不混页 |
| BR-002 | A4 纵向一页最多 2 张，横向一页最多 4 张 |
| BR-003 | 每组的最后一页如果不足满页，剩余位置留空，不强制补满 |
| BR-004 | 导出 PDF 中不加任何辅助信息（页眉、页码、发票标注等） |
| BR-005 | 发票内容（图片/PDF 页面）按比例缩放适配格子，不被拉伸变形 |
| BR-006 | 用户只能导出自己（当前登录用户）的发票 |
| BR-007 | 批次导出时，导出该批次关联的所有发票（含替票发票） |
| BR-008 | 同一次导出中，不同类型的发票可分别指定不同的排版模式（纵向/横向） |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 批次详情页导出 PDF — 统一纵向排版
  - **Given** 用户已登录，批次详情页中该批次关联了 4 张发票（2 张增值税电子发票 + 2 张高铁票）
  - **When** 用户在导出弹窗中将两种类型都设为"纵向（一页2张）"，点击确认导出
  - **Then** 后端返回一个 PDF 文件下载，PDF 共 2 页：第 1 页上下排列 2 张增值税发票，第 2 页上下排列 2 张高铁票

- [ ] **AC-002**: 批次详情页导出 PDF — 不同类型不同排版
  - **Given** 用户已登录，批次关联了 5 张发票（3 张增值税发票 + 2 张滴滴发票）
  - **When** 用户在导出弹窗中将增值税发票设为"横向（一页4张）"，滴滴发票设为"纵向（一页2张）"，点击确认导出
  - **Then** 后端返回一个 PDF：第 1 页 2×2 排列 3 张增值税发票（第 4 个位置留空），第 2 页上下排列 2 张滴滴发票

- [ ] **AC-003**: 同类型发票集中排放
  - **Given** 用户导出 6 张发票：3 张增值税发票 + 3 张高铁票
  - **When** 用户将两种类型都设为纵向排版，确认导出
  - **Then** PDF 第 1-2 页连续排放 3 张增值税发票（第 2 页只有 1 张，下半页留空），第 3 页排放高铁票，不同类型不混在同一页

- [ ] **AC-004**: 单种类型不足满页留空处理
  - **Given** 导出 1 张增值税发票 + 1 张高铁票，选择纵向排版
  - **When** 生成 PDF
  - **Then** 第 1 页上半页放置增值税发票，下半页留空；第 2 页上半页放置高铁票，下半页留空

- [ ] **AC-005**: PDF 文件原样输出无辅助信息
  - **Given** 导出任意发票
  - **When** 打开生成的 PDF
  - **Then** PDF 内容只有发票本身，没有任何页眉、页脚、页码、发票号码标注、导出时间等额外文字或标记

- [ ] **AC-006**: 发票列表页自由勾选导出
  - **Given** 用户在发票列表页，有多张 confirmed 状态的发票
  - **When** 用户勾选 3 张发票，点击"导出 PDF"，选择横向排版后确认
  - **Then** 后端返回 PDF，只包含用户勾选的 3 张发票，横向排版第 1 页左上/右上/左下 3 个格子有发票，右下留空

- [ ] **AC-007**: PDF/图片发票统一处理
  - **Given** 导出的发票中既有 PDF 电子发票，也有 JPG/PNG 格式图片发票
  - **When** 生成 PDF
  - **Then** 两种格式的发票均按比例缩放适配到 A4 页面对应格子中，内容完整可见

- [ ] **AC-018**: 不同类型分别选择不同排版
  - **Given** 用户导出 4 张发票：2 张增值税发票 + 2 张高铁发票
  - **When** 用户在导出弹窗中将增值税发票设为"横向（一页4张）"，高铁发票设为"纵向（一页2张）"，点击确认导出
  - **Then** PDF 第 1 页 2×2 排列 2 张增值税发票（后 2 格留空），第 2 页上下排列 2 张高铁发票

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-008**: 导出空批次
  - **Given** 批次中没有任何发票
  - **When** 用户在批次详情页点击"导出 PDF"
  - **Then** "导出 PDF"按钮置灰不可点击，或点击后提示"该批次暂无发票可导出"

- [ ] **AC-009**: 导出单张发票
  - **Given** 用户选择导出 1 张发票
  - **When** 选择纵向排版导出
  - **Then** PDF 只有 1 页，上半页放置发票，下半页留空

- [ ] **AC-010**: 导出大量发票
  - **Given** 用户导出 50 张同类型发票，选择横向排版
  - **When** 生成 PDF
  - **Then** PDF 共 13 页（50÷4=12.5，取整 13 页），最后一页只有 2 张（后 2 格留空），所有发票按比例正确缩放

- [ ] **AC-011**: 发票文件不存在
  - **Given** 导出的发票中某张发票的原始文件在服务器上已被删除
  - **When** 生成 PDF
  - **Then** 该发票位置显示占位提示（如"文件不可用"），不影响其他发票的正常导出

- [ ] **AC-012**: 未登录访问导出接口
  - **Given** 用户未登录
  - **When** 直接请求导出 API
  - **Then** 返回 401 未授权错误

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-013**: 按类型分组排放 ← BR-001
  - **Given** 导出发票包含 3 种不同类型
  - **When** 生成 PDF
  - **Then** 同一类型的发票在 PDF 中连续出现，每页内不混有两种类型

- [ ] **AC-014**: 排版容量上限 ← BR-002
  - **Given** 用户选择横向排版导出 5 张发票
  - **When** 生成 PDF
  - **Then** 第 1 页放 4 张（2×2），第 2 页放 1 张（仅左上位置），页内不叠加超过容量上限

- [ ] **AC-015**: 不足满页留空 ← BR-003
  - **Given** 用户选择纵向排版导出 3 张发票
  - **When** 生成 PDF
  - **Then** 第 1 页放 2 张（上下排列），第 2 页放 1 张（上半页），下半页留空

- [ ] **AC-016**: 数据隔离 ← BR-006
  - **Given** 用户 A 和用户 B 各有发票
  - **When** 用户 A 导出 PDF
  - **Then** 导出结果只包含用户 A 自己的发票

- [ ] **AC-017**: 批次导出含替票 ← BR-007
  - **Given** 批次内有一张替票发票（is_substitute=true）
  - **When** 从批次详情页导出 PDF
  - **Then** 该替票发票被包含在导出结果中

- [ ] **AC-019**: 按类型设定不同排版 ← BR-008
  - **Given** 导出 3 种不同类型的发票，分别各有若干张
  - **When** 用户在导出弹窗中为每种类型分别选择不同的排版模式
  - **Then** 每种类型按各自选定的排版模式生成页面，互不影响

## 6. 范围界定总结

### 本次做

| 功能 | 优先级 |
|------|--------|
| 后端按类型分组 + 逐类型独立排版生成 PDF | P0 |
| PDF/图片统一缩放处理 | P0 |
| 导出 API：POST /api/exports/invoice-pdf | P0 |
| 导出 API：POST /api/batches/{id}/export-invoice-pdf | P0 |
| 前端排版选择弹窗（按类型分组 + 逐项选择排版 + 统一设置快捷操作） | P0 |
| 批次详情页"导出 PDF"入口 | P0 |
| 发票列表页勾选导出入口 | P0 |
| 导出按钮加载状态防重复 | P0 |

### 本次不做

| 功能 | 理由 |
|------|------|
| 页码/页眉/水印等辅助信息 | 用户要求保持发票原始面貌 |
| 在线 PDF 预览 | 浏览器原生 PDF 查看已满足需求 |
| 费用报销单合并 | Phase 2.5-2.6 单独处理 |
| 多个批次合并导出 | 当前无此场景需求 |
