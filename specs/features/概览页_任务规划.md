# 概览页 (Dashboard) — 任务规划

## 阶段划分概览

```
Phase 1: 后端统计接口（基础设施）
  └── Task 1.1 Schema 定义
  └── Task 1.2 Service 层统计逻辑
  └── Task 1.3 API 路由 + 注册
         ↓
Phase 2: 概览页前端展示（垂直切片）
  └── Task 2.1 前端类型 + API 层
  └── Task 2.2 Store 状态管理
  └── Task 2.3 页面组件实现
```

**依赖关系**：Phase 2 依赖 Phase 1（后端 API 就绪后前端才能调通）

**阶段完成标准**：
- Phase 1 完成后：`GET /api/dashboard/stats` 返回正确的统计数据
- Phase 2 完成后：用户登录后访问 `/dashboard`，看到完整的概览页

---

## Phase 1: 后端统计接口（基础设施层）

### Task 1.1: 定义 Dashboard 的 Pydantic Schema

| 属性 | 值 |
|------|-----|
| **通俗解释** | 定义后端返回统计数据的"信封格式"——前端拿到什么字段、什么类型，都由这里决定 |
| **输入** | 技术方案 2.2 节的 Schema 定义 |
| **输出** | `server/app/schemas/dashboard.py` |
| **关联 AC** | AC-001 ~ AC-009（所有 AC 的数据载体） |
| **技术方案章节** | 2.2 Schema 定义 |

**验证标准**：
1. `RecentBatchItem` 包含 id、department、period_start、period_end、report_date、reporter、total_amount、status、invoice_count、created_at 字段，类型与技术方案一致
2. `DashboardStatsResponse` 包含 pending_invoice_count(int)、monthly_total_amount(float)、active_batch_count(int)、recent_batches(list[RecentBatchItem])
3. 两个类均有 `model_config = {"from_attributes": True}`
4. `pytest` 导入无报错

---

### Task 1.2: 实现 Dashboard 统计查询 Service

| 属性 | 值 |
|------|-----|
| **通俗解释** | 每次用户打开概览页，后端执行 4 条 SQL 查询：待处理发票有几张、本月报销了多少钱、还有几个批次没归档、最近创建了哪 5 个批次 |
| **输入** | 技术方案 2.3 节的 Service 逻辑 |
| **输出** | `server/app/services/dashboard_service.py` |
| **关联 AC** | AC-001、AC-002、AC-003、AC-004、AC-008、AC-009 |
| **技术方案章节** | 2.3 Service 层 |

**验证标准**：
1. `get_dashboard_stats(db, user_id)` 返回 `DashboardStatsResponse` 实例
2. **待处理发票数 (AC-001)**：向 invoices 表插入 3 条 status='pending'、2 条 status='confirmed'（同 user_id）→ pending_invoice_count = 3
3. **本月报销总额 (AC-002 + AC-008)**：向 invoices 表插入本月 2 条 confirmed 发票（金额 100.50 + 200.00）、上月 1 条 confirmed 发票（金额 300.00）→ monthly_total_amount = 300.50；已软删除的本月 confirmed 发票不纳入统计
4. **进行中批次数 (AC-003)**：非软删除的批次才计数
5. **最近批次 (AC-004 + AC-009)**：有 6 个批次时，只返回 5 个，按 created_at 降序排列

---

### Task 1.3: 实现 Dashboard API 路由并注册

| 属性 | 值 |
|------|-----|
| **通俗解释** | 把统计查询功能"挂"到网上——前端调用 `GET /api/dashboard/stats` 就能拿到数据 |
| **输入** | 技术方案 2.4 + 2.5 节的 API 路由和注册 |
| **输出** | `server/app/api/dashboard.py`、修改 `server/app/api/router.py` |
| **关联 AC** | AC-001 ~ AC-009 |
| **技术方案章节** | 2.4 API 路由、2.5 注册路由 |

**验证标准**：
1. `GET /api/dashboard/stats` 带有效 JWT Token → 200，返回 JSON 包含 pending_invoice_count、monthly_total_amount、active_batch_count、recent_batches 字段
2. 不传 Token → 401
3. Swagger 文档可访问 `/docs`，能看到 dashboard 分组下的 `/api/dashboard/stats` 接口

## Phase 2: 概览页前端展示（垂直切片）

| 属性 | 值 |
|------|-----|
| **通俗解释** | 前端知道后端返回的数据长什么样，知道怎么向后端发请求拿数据 |

| **技术方案章节** | 3.1 类型定义、3.2 API 层 |

**验证标准**：
1. `DashboardStats` 类型包含 pending_invoice_count: number、monthly_total_amount: number、active_batch_count: number、recent_batches: RecentBatchItem[]
2. `dashboardApi.getStats()` 调用 `GET /api/dashboard/stats` 并返回 `DashboardStats`
3. 后端返回的 json 能正确映射到前端类型

---

### Task 2.2: 实现 Dashboard Store

| 属性 | 值 |
|------|-----|
| **通俗解释** | 管理概览页数据的"内存缓存"——页面打开时去后端拉数据，加载中/加载失败时记录状态，方便页面展示不同状态 |
| **输入** | 技术方案 3.3 节 |
| **输出** | `web/src/stores/dashboardStore.ts` |
| **关联 AC** | AC-007（加载失败降级） |
| **技术方案章节** | 3.3 状态管理 |

**验证标准**：
1. `fetchStats()` 调用 `dashboardApi.getStats()`，成功后 `stats` 字段赋值为返回数据，`loading = false`
2. `fetchStats()` 请求失败时→ `error = true`、`loading = false`（AC-007 降级基础）
3. `fetchStats()` 请求中→ `loading = true`

---

### Task 2.3: 实现概览页页面组件

| 属性 | 值 |
|------|-----|
| **通俗解释** | 用户登录后看到的第一个页面——左边三张数字卡片告诉你发票和批次概况，中间列出最近的批次，底部是三个快捷入口按钮。新用户看到引导文案引导开始使用 |
| **输入** | 技术方案 3.4 节 + 需求文档 6. 界面布局示意 |
| **输出** | 重写 `web/src/pages/DashboardPage.tsx` |
| **关联 AC** | AC-001 ~ AC-007（全部前端可见行为） |
| **技术方案章节** | 3.4 页面组件 |

**验证标准**：
1. **统计卡片 (AC-001~003)**：页面加载后显示 3 张统计卡片——"待处理发票数"、"本月报销总额"、"进行中批次数"
2. **金额格式化 (AC-002)**：`monthly_total_amount` 显示为 `¥3,280.50` 格式，精确到两位小数
3. **待处理卡片跳转 (AC-001)**：点击"待处理发票数"卡片→跳转到 `/invoices?tab=pending`
4. **进行中卡片跳转 (AC-003)**：点击"进行中批次数"卡片→跳转到 `/batches`
5. **最近批次列表 (AC-004)**：有批次时，展示最近 5 条，每条显示批次名称、金额合计、创建日期；点击某条→跳转 `/batches/{id}`
6. **快捷操作 (AC-005)**：展示"上传发票"、"创建批次"、"导出台账"三个按钮/卡片，点击分别跳转 `/invoices`、`/batches`、`/batches`
7. **新用户空状态 (AC-006)**：pending_invoice_count=0 且 active_batch_count=0 时，展示空状态引导卡片，含友好文案和"上传发票"按钮
8. **加载失败降级 (AC-007)**：error=true 时，统计卡片数字显示 `——`，最近批次和快捷操作区域不阻塞展示
9. **加载中状态**：loading=true 时，统计卡片区域显示加载骨架屏或文字"加载中..."
10. **金额为 0 且非空状态**：有批次但 total_amount=0，显示 `¥0.00`
