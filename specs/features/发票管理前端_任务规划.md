# 发票管理前端 — 任务规划

## 1. 切片策略

采用**垂直切片**策略，每个切片对应一个独立的用户行为，穿透所有技术层（类型 → API → Store → UI 组件 → 页面集成）。

```
Stage 0: 基础设施 ──────────────── 所有切片的基石
  ├── T-00 类型扩展
  ├── T-01 API 层封装
  ├── T-02 Store 层实现
  ├── T-03 常量+工具函数
  ├── T-04 Modal 组件
  └── T-05 Table 组件
                │
                ├──────────────────────────────────────────┐
                ▼                                          ▼
Stage 1: 发票上传 ──────────────    Stage 2: 发票列表浏览 ────
  ├── T-06 InvoiceUploader           ├── T-08 InvoiceTabs+Filter
  └── T-07 EmptyState                ├── T-09 InvoiceCard
                                     ├── T-10 ListView+CardView
                                     ├── T-11 Pagination
                                     └── T-12 InvoicesPage 集成
                                              │
                                              ├──────────────────┐
                                              ▼                  ▼
                                     Stage 3: 详情与编辑    Stage 4: 删除&回收站
                                       ├── T-13 Modal+Editor   ├── T-15 TrashModal
                                       └── T-14 页面集成       └── T-16 删除流程
```

**依赖关系**：
- Stage 1、2、3、4 全部依赖 Stage 0
- Stage 3 依赖 Stage 2（需从列表打开详情）
- Stage 4 依赖 Stage 2（需从列表/详情执行删除）
- Stage 3 和 Stage 4 之间无依赖，可串行或并行

---

## 2. 执行顺序建议

| 顺序 | 阶段 | 说明 | 预估工时 | 可验证行为 |
|------|------|------|---------|-----------|
| 1 | Stage 0: 基础设施 | 必须最先完成 | 40-50 分钟 | 无用户可见行为，但所有后续切片依赖 |
| 2 | Stage 1: 发票上传 | 可独立验证 | 25-35 分钟 | 用户可拖拽/选文件，预览后上传 |
| 3 | Stage 2: 发票列表浏览 | 依赖 Stage 1（有数据才能验证列表） | 50-70 分钟 | 用户可看到已上传发票，切换Tab/视图/筛选/翻页 |
| 4 | Stage 3: 发票详情查看与编辑 | 依赖 Stage 2 | 40-50 分钟 | 用户可打开详情、查看文件、编辑字段、确认入库 |
| 5 | Stage 4: 删除与回收站 | 依赖 Stage 2 | 25-35 分钟 | 用户可删除发票、查看回收站、恢复发票 |

**总预估工时**：180-240 分钟
