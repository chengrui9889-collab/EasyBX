#### Task-004：实现配置 CRUD API

**通俗解释**：提供配置的增删改查接口，让前端可以操作配置数据。

**技术方案章节**：3.1, 3.3

**涉及 AC**：AC-001, AC-002, AC-003, AC-004, AC-009, AC-010, AC-011, AC-012, AC-013, AC-014, AC-015

**依赖**：Task-003

**验证标准**：

**GET /api/configs**：
- 返回当前用户的配置列表，按 created_at 排序
- 无配置时返回 `{items: [], total: 0}`

**POST /api/configs**：
- 传入 `{name: "程瑞-产教融合", reporter: "程瑞", payee: "程瑞"}` → 返回 201，body 包含完整配置数据，is_default=true（第一个配置）
- 传入 `{name: ""}` → 返回 400，detail.message = "name 不能为空"
- 传入已存在的 name → 返回 400，detail.code = "CONFIG_NAME_EXISTS"

**PUT /api/configs/{id}**：
- 传入 `{reporter: "张三"}` → 返回 200，配置更新成功
- 传入已存在的 name → 返回 400，detail.code = "CONFIG_NAME_EXISTS"
- 传入不存在的 id → 返回 404

**DELETE /api/configs/{id}**：
- 删除成功 → 返回 `{deleted: true}`
- 删除不存在的 id → 返回 404
- 删除默认配置后，剩余第一个配置的 is_default 自动变为 true

**POST /api/configs/{id}/set-default**：
- 设置成功 → 返回 200，该配置 is_default=true，其他配置 is_default=false
- 传入不存在的 id → 返回 404
