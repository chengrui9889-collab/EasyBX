# 概览页 (Dashboard) — 技术方案设计

## 1. 设计概览

### 1.1 目标

为已登录用户提供概览页（工作台），展示：
- 三个统计卡片：待处理发票数、本月报销总额、进行中批次数
- 最近 5 条报销批次列表
- 快捷操作入口：上传发票、创建批次、导出台账
- 新用户空状态引导

### 2.3 Service 层

文件：`server/app/services/dashboard_service.py`

```python
from datetime import date
from sqlalchemy import func
from sqlalchemy.orm import Session

from app.models.invoice import Invoice
from app.models.batch import BatchInvoice, ReimbursementBatch
from app.schemas.dashboard import DashboardStatsResponse, RecentBatchItem


def get_dashboard_stats(db: Session, user_id: int) -> DashboardStatsResponse:
    today = date.today()
    first_day_of_month = today.replace(day=1)

    # AC-001: 待处理发票数 — status='pending' 且未删除
    pending_count = (
        db.query(func.count(Invoice.id))
        .filter(
            Invoice.user_id == user_id,
            Invoice.status == "pending",
            Invoice.deleted_at.is_(None),
        )
        .scalar()
        or 0
    )

    # AC-002 + AC-008: 本月报销总额 — status='confirmed' 且当月、未删除
    monthly_total = (
        db.query(func.coalesce(func.sum(Invoice.amount), 0.0))
        .filter(
            Invoice.user_id == user_id,
            Invoice.status == "confirmed",
            Invoice.deleted_at.is_(None),
            Invoice.invoice_date >= first_day_of_month,
            Invoice.invoice_date <= today,
        )
        .scalar()
        or 0.0
    )

    # AC-003: 进行中批次数 — 未删除的批次
    active_batch_count = (
        db.query(func.count(ReimbursementBatch.id))
        .filter(ReimbursementBatch.user_id == user_id)
        .filter(ReimbursementBatch.deleted_at.is_(None))  # 如果需要软删除
        .scalar()
        or 0
    )

    # AC-004 + AC-009: 最近 5 条批次 — 按 created_at 倒序
    recent_batches_query = (
        db.query(ReimbursementBatch)
        .filter(ReimbursementBatch.user_id == user_id)
        .order_by(ReimbursementBatch.created_at.desc())
        .limit(5)
        .all()
    )

    recent_batches = []
    for batch in recent_batches_query:
        invoice_count = (
            db.query(BatchInvoice)
            .filter(BatchInvoice.batch_id == batch.id)
            .count()
        )
        recent_batches.append(RecentBatchItem(
            id=batch.id,
            department=batch.department,
            period_start=batch.period_start,
            period_end=batch.period_end,
            report_date=batch.report_date,
            reporter=batch.reporter,
            total_amount=batch.total_amount,
            status=batch.status,
            invoice_count=invoice_count,
            created_at=batch.created_at,
        ))

    return DashboardStatsResponse(
        pending_invoice_count=pending_count,
        monthly_total_amount=round(monthly_total, 2),
        active_batch_count=active_batch_count,
        recent_batches=recent_batches,
    )
```

### 2.4 API 路由

文件：`server/app/api/dashboard.py`

```python
from fastapi import APIRouter, Depends
from sqlalchemy.orm import Session

from app.database import get_db
from app.dependencies import get_current_user
from app.models.user import User
from app.schemas.dashboard import DashboardStatsResponse
from app.services.dashboard_service import get_dashboard_stats

router = APIRouter(prefix="/dashboard", tags=["dashboard"])


@router.get("/stats", response_model=DashboardStatsResponse)
async def api_dashboard_stats(
    current_user: User = Depends(get_current_user),
    db: Session = Depends(get_db),
):
    return get_dashboard_stats(db, current_user.id)
```

```typescript
import { apiClient } from './client';
import type { DashboardStats } from '@/types/dashboard';

export const dashboardApi = {
  getStats: async (): Promise<DashboardStats> => {
    const r = await apiClient.get('/dashboard/stats');
    return r.data;
  },
};
```

### 3.3 状态管理

文件：`web/src/stores/dashboardStore.ts`

```typescript
import { create } from 'zustand';
import type { DashboardStats } from '@/types/dashboard';
import { dashboardApi } from '@/api/dashboard';

interface DashboardState {
  stats: DashboardStats | null;
  loading: boolean;
  error: boolean;
  fetchStats: () => Promise<void>;
}

export const useDashboardStore = create<DashboardState>((set) => ({
  stats: null,
  loading: false,
  error: false,

  fetchStats: async () => {
    set({ loading: true, error: false });
    try {
      const stats = await dashboardApi.getStats();
      set({ stats, loading: false });
    } catch {
      // AC-007: 加载失败降级，不阻断渲染
      set({ error: true, loading: false });
    }
  },
}));
```

### 3.4 页面组件

文件：`web/src/pages/DashboardPage.tsx`

组件结构：

```
DashboardPage
├── StatsCards（统计卡片区域）
│   ├── StatCard: 待处理发票数 → 点击跳转 /invoices?tab=pending
│   ├── StatCard: 本月报销总额
│   └── StatCard: 进行中批次数 → 点击跳转 /batches
│
├── EmptyStateGuide（AC-006: 新用户空状态，当无数据时展示）
│
├── RecentBatches（最近批次区域 → 仅当有批次时展示）
│   ├── RecentBatchItem × 1~5（可点击跳转批次详情）
│   └── "查看更多"链接 → 跳转 /batches
│
└── QuickActions（快捷操作区域）
    ├── "上传发票" → 跳转 /invoices
    ├── "创建批次" → 跳转 /batches
    └── "导出台账" → 跳转 /batches
```

**关键交互细节**：

| AC | 交互行为 |
|----|---------|
| AC-001 | 待处理发票数卡片点击后，跳转 `/invoices`，前端在 invoiceStore 设置 activeTab='pending' |
| AC-004 | 最近批次点击后，跳转 `/batches/{id}` |
| AC-006 | 空状态判断：`pending_invoice_count === 0 && active_batch_count === 0` |
| AC-007 | `error === true` 时，统计数字显示 `——`，最近批次和快捷操作正常渲染 |

## 4. 核心逻辑与异常处理

### 4.1 错误场景覆盖

| 场景 | 后端行为 | 前端行为 | 关联 AC |
|------|---------|---------|---------|
| DB 连接失败 | HTTP 500 | 统计卡片显示 `——`，快捷操作正常展示 | AC-007 |
| 无数据（新用户） | 返回 0 计数和空列表 | 展示空状态引导卡片 | AC-006 |
| 统计接口超时 | HTTP 503 | 统计卡片显示 `——` | AC-007 |
| 用户未认证 | HTTP 401 | 前端 axios 拦截器自动跳转登录页 | — |

### 4.2 业务规则

| 规则 | 实现方式 | 关联 AC |
|------|---------|---------|
| 本月范围 = 当前自然月（1日至当日） | `first_day_of_month = today.replace(day=1)` | AC-008 |
| 不包含已软删除发票 | `Invoice.deleted_at.is_(None)` | AC-008 |
| 最近批次按 created_at 倒序 | `.order_by(ReimbursementBatch.created_at.desc())` | AC-009 |
| 金额精确到分 | `round(monthly_total, 2)` | AC-002 |
