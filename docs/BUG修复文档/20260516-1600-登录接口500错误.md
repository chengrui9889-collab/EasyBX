# BUG 修复报告：登录接口返回 500 错误

**日期**: 2026-05-16
**严重程度**: 高（阻断用户登录）
**影响范围**: 所有需要登录的前端操作

---

## 问题现象

浏览器访问前端页面（`http://localhost:5180`）时，点击登录按钮后报错：

```
POST http://localhost:5180/api/auth/login 500 (Internal Server Error)
```

用户无法登录系统。

---

## 复现步骤

1. 启动前端开发服务器（Vite，端口 5180）
2. **未启动后端服务器**（或后端在其他端口运行）
3. 打开浏览器访问 `http://localhost:5180`
4. 输入用户名密码点击登录
5. 浏览器控制台显示 500 错误

---

## 根因分析

**直接原因**：后端服务器未在端口 220 上运行。

**链路分析**：

```
浏览器 ──POST /api/auth/login──▶ Vite Dev Server (5180)
                                      │
                                      │ proxy: /api → http://localhost:220
                                      ▼
                                 Backend Server (220)  ← 未运行！
                                      │
                                      ▼
                                 连接被拒绝 → Vite 返回 500
```

1. `web/vite.config.ts` 中配置了代理：`/api` 请求转发到 `http://localhost:220`
2. `server/.env` 中配置 `SERVER_PORT=0220`，期望后端在端口 220
3. 但后端未在 220 端口启动，Vite 代理无法连接后端，返回 500

**验证**：
- 用 TestClient 直接测试登录 API → 正常返回 401（用户不存在，逻辑正确）
- 用 uvicorn 在端口 220 启动后端后 → 请求返回 200 OK

---

## 修复方案

**修复**：在正确端口 220 启动后端服务器

```bash
cd server
python -m uvicorn main:app --port 220 --reload
```

**当前状态**：已启动，服务运行正常。

---

## 预防措施

建议在项目中添加一个统一的启动脚本（如 `start.bat` 或 `package.json` scripts），确保前后端同时以正确端口启动，避免忘记启动某端或端口不匹配。
